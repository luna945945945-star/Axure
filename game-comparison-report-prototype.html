📋 业务规则 & 验收清单 配合原型阅读 · 开发 / 测试必看
① 商户名称
  • 必填，单选下拉
  • 数据源：当前登录用户有权限的商户列表
  • 切换商户时 → 清空：厂商列表 + 对配关系 + 模板状态
② 游戏子类
  • 单选下拉，默认「全部」
  • 选项：全部 / lottery / video / Slot / Fish / sport / ChessCard / small game / WinGo
  • 切换子类 → 清空对配关系（子游戏池范围变了）
③ 游戏厂商（含独立日期）
  • 多选，最少 1 家、最多 4 家
  • 同一厂商不可重复添加；下拉中已选项灰显 + 标「✓ 已选」
  • 厂商列表按所选商户过滤
  • 每家厂商有独立日期范围（chip 内可编辑）
  • 日期校验：起 ≤ 止；最大跨度 90 天；不可超今日；不早于 2024-01-01
  • 添加厂商 → 对配关系表每行新增该厂商列（默认未选）
  • 删除厂商 → 对配关系表同步删除该厂商列
  • 修改厂商日期 → 该厂商所有数据列重新查询
  • 厂商数 = 1 时，× 禁用，hover 提示「至少保留 1 家厂商」
  • 厂商数 = 4 时，「+ 添加厂商」按钮禁用
④ 对配关系
  • 列结构：# / 对配名 / 厂商A·子游戏 / 厂商B·子游戏 / ... / ×
  • 查询前至少 1 行有效对配，最多 20 行
  • 对配名必填，长度 1-30 字符，同一查询内不可重复
  • 每行每个厂商列都必须选子游戏，否则该行不算"有效"
  • 同一厂商的同一子游戏不可在多行中重复选
  • 「+ 添加对配」点击后表底插入新空白行
  • 删除行后自动重排序号；删最后一行需二次确认
  • 子游戏下拉选项 = (商户 + 厂商 + 游戏子类) 三者过滤
  • 子游戏展示格式：厂商编码 — 游戏中文名
  • 切换子类后，所有对配行的子游戏选择清空
⑤ 载入模板
  • 下拉面板列出当前用户保存的全部模板
  • 每条展示：模板名 / 对配数 / 涉及厂商 / 上次使用时间
  • 顶部搜索框按模板名实时过滤
  • 点击模板触发二次确认："载入会覆盖当前对配，确定？"
  • 确认后填充：对配列表 + 厂商列表 + 各厂商日期范围
  • 模板含当前商户未接入的厂商 → 跳过并提示
  • 模板按用户私有；列表为空时显示空状态
  • 支持重命名 ✏（弹窗输入）和删除 ×（二次确认）
⑥ 保存为模板
  • 对话框输入：模板名（必填）+ 备注（可选）
  • 模板名 1-30 字符，备注 ≤ 200 字符
  • 保存前校验：至少 1 行有效对配（对配名非空 + 每厂商列已选）
  • 同名模板已存在 → 弹「同名模板已存在，是否覆盖？」
  • 单用户模板上限 50 个
  • 模板内容含：对配关系 + 厂商列表 + 各厂商日期范围 + 对配名
⑦ 查询 / 重置 / 导出
  • 查询前置：商户已选 + 厂商 ≥ 2 + 对配 ≥ 1 行有效
  • 任一条件不满足 → 按钮禁用 + Tooltip 显示原因
  • 查询中按钮 Loading，防重复点击
  • 重置需二次确认，恢复默认
  • 导出仅在已生成报表后可用
  • 导出 Excel = 当前所有游戏卡片表格（含合计行）
  • 文件名：游戏比对报表_{商户}_{当前日期}.xlsx
⑧ 游戏比对卡片
  • 一张卡片 = 一行对配关系
  • 卡片头：对配名 + 各厂商色标签（不展示日期，日期已在筛选区）
  • 每段 11 列：日期 / 投注人数 / 投注笔数 / 投注金额 / 派彩金额 / 输赢额 / 盈利率 / 有效投注比 / 人均投注笔数 / 人均投注金额 / 平均每笔投注额
  • 段头颜色：PG 蓝 / UP 橙 / JILI 紫 / EVO 青 / PT 红 / MG 粉
  • 数据行无底色（白）；合计行橙底加粗 + 顶部 2px 橙线
  • 盈利率：正绿加粗 / 负红加粗 / 0 灰
  • 输赢额：正绿 / 负红
  • PG 段与 UP 段之间留 30px 缓冲列
⑨ 字段公式
字段公式
投注人数当日去重玩家数
投注笔数COUNT(投注订单)
投注金额SUM(BetAmount)
派彩金额SUM(WinAmount)
输赢额投注金额 − 派彩金额
盈利率输赢额 / 投注金额 × 100%
有效投注比有效投注金额 / 总投注金额 × 100%
人均投注笔数投注笔数 / 投注人数
人均投注金额投注金额 / 投注人数
平均每笔投注额投注金额 / 投注笔数
合计行特别说明
人数/笔数/金额/派彩/输赢期内求和
盈利率总输赢 / 总投注
有效投注比按金额加权 = Σ(有效投注金额) / Σ(总投注金额) — 不是简单平均
人均 / 笔均基于求和结果计算
⑩ 核心规则速查表
操作限制

切换子类清空对配行的子游戏选择
