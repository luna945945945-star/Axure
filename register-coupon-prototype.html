一、名词术语
术语定义
充值优惠券码可拼接进注册链接 / 渠道域名的字符码，携带券规格（比例 / 上限 / 门槛 / 已领取券有效期 / 打码量 / 适用充值大类 / 白名单），用户注册命中即发券。
适用渠道券码的发放场景，共 4 类：渠道注册码、
人工充值、
代理邀请链接、
代理红包码。
券码 × 代理粒度同一券码授权给多个代理时，每个代理一行记录，可按代理ID 单独追溯发放进度。
手机号白名单注册场景专用号码池，仅对手机号注册用户命中，用于定向拉新与防水军。
适用充值大类券可用于哪些充值通道大类；
充值时按本单大类校验，不符则该券置灰。
券 vs 通道赠金关系券规格的一项配置，决定用券充值时券赠金与通道赠金谁生效、是否相加，三选一：互斥·取高（两者只取金额高的一方，平局取券，券落败则不消耗、退回券包）/ 叠加（两者全给、相加）/ 互斥·优先券（无条件只给券赠金、完全不比通道）。计算口径见「3.9 赠金计算规则（券 vs 通道赠金）」。

3.9 赠金计算规则（券 vs 通道赠金）

本节是「券赠金」与「通道赠金」的唯一口径，#31 / #16 / #20 三份需求共用；其余章节与业务规则只引用本节、不重复。

业务流程：用户用券充值时，实际到账 = 实付金额 ＋ 赠金 − 手续费。「赠金」由两部分竞争或叠加得到——券赠金（这张券带来的）与通道赠金（充值方式自带的活动奖励）；
二者谁生效、是否相加，由券规格里配置的赠金关系决定（三选一，运营建券时设定，代理 / 会员不可改）。

字段定义：

概念说明
实付金额用户本次实际充值金额；
券门槛、券比例、手续费均以它为基数（不以到账额为基数）
券赠金实付金额按券比例计算，不超过券上限；
未达券门槛则该券不可用
通道赠金充值方式自带的活动奖励，含三类：充值通道赠金 / VIP等级充值奖励 / 快捷金额充值奖励
手续费按实付金额的固定费率扣取
打码量赠金到账后、提现前需完成的流水倍数；
券赠金与通道赠金各自按自己的倍数计算
取整规则券赠金、通道赠金、手续费均向下取整为整数；
展示统一保留 2 位小数（如 50 显示为 50.00），各自单独计算

三种赠金关系：

赠金关系含义券是否消耗到账明细
互斥·取高
（默认）		券赠金与三类通道奖励中只发金额最高的一个；
金额相等时优先发券		券胜出 → 消耗；
券未胜出 → 不消耗、退回券包，不计入「已使用」		一行「奖励」＋来源标注
叠加券赠金与三类通道奖励全部发放、金额相加消耗两行：「充值奖励」（三类通道合计）＋「充值优惠券彩金」；
两行金额相加 = 总赠金
互斥·优先券无条件只发券赠金，不与通道比较（即使通道更高）消耗一行「奖励」＋来源标注

来源标注：明细为一行时（取高 / 优先券）必须标注赠金来源，让会员看清钱从哪来、券有没有被扣：

  • 券胜出：奖励 +₹50（充值优惠券 10%，高于通道 ₹45）
  • 券未胜出：奖励 +₹90（VIP等级奖励 高于券 ₹50；券未使用、已退回）

四、业务规则
BR-01
4 类适用渠道：渠道注册码（渠道域名注册送）/ 人工充值 / 代理邀请链接（代理拼链接）/ 代理红包码（发给已注册会员）。
BR-02
分配代理仅「代理邀请链接 / 代理红包码」需要（授权哪些代理可推广 / 发放）；
渠道注册码、人工充值无需。
BR-03
手机号白名单归位：白名单属券规格，渠道注册码 / 代理邀请链接均在「建券弹窗」设定；
渠道报表引用该券时，白名单按券规格固定为只读，不再在渠道配置单独设。
BR-04
适用充值大类必勾：至少勾 1 个（全选＝全部大类可用）；
不再是「不勾＝全部」。充值页按本单大类校验，不符则该券置灰。
BR-05
券 vs 通道赠金关系：互斥·取高（默认）/ 叠加 / 互斥·优先券，三选一。计算口径见「3.9 赠金计算规则（券 vs 通道赠金）」，本条不另立算式。
BR-06
券码 × 代理粒度：一码多代理＝多行，可按代理ID 追溯单代理发放进度。「券码发放上限」为每代理独立配额——分配给 N 个代理时，每个代理各得「上限」次、
各自计数、
互不占用；
某代理达到自己的上限 → 仅该代理行置为「已用完」（其链接注册不再发券），其他代理不受影响；
全部代理行均用完 → 券码整体「已用完」。0＝不限，未设则仅按券码有效期控制。手机号白名单则相反，为券级共享：券绑定的白名单由所有被分配代理共用同一批号码（不按代理拆分）；
名单内号码通过哪个代理的注册链接（invitationCode 归属）注册，发券与业绩即归因该代理，同一号码全券仍只可领 1 次（先到先得，跨代理不重复发）。
BR-07
充值优惠券到账逻辑复用现有充值优惠券系统：发出的券进「我的充值优惠券」，充值使用与到账口径与现有充值优惠券系统一致。
BR-08
白名单券码不在注册页展示：注册页无法在输入手机号前判定名单归属，故绑白名单的券码注册页不展示券区（不锁态、不占位）；
命中名单的手机号注册成功后券自动入袋 + 首次登录弹窗告知——弹窗直接复用 #20「弹窗领取」组件（同一套 UI，不另设计）；
但语义按 #20 口径「注册得券一律自动入袋」：券在注册成功时已入袋，弹窗仅为到账告知，不点 / 未弹到也不影响券归属，无「领取截止」、无额度退回；
弹一次即止，站内信兜底；
未命中不拦截注册，仅不发券；
前端不提供白名单实时查询接口，避免手机号在名单内与否被枚举探测。
BR-09
代理选券时必须标注白名单归属：代理端「邀请海报」选券下拉需逐条注明「🔒限白名单」/「不限名单」；
选中绑白名单的券后，页面显式提示所限名单名称及后果——名单外的人注册不发券、落地页不展示券（对方全程无感知），发给名单外＝白发。与「即将到期」提示可并存（同一张券可能既绑名单又临近到期）。目的：避免代理误把定向券当通用券群发。
五、状态与判定
5.1 券码状态
  • 进行中：活动时段内、未停用，可正常发券。
  • 已用完：某代理发放达到其独立配额 → 仅该代理行「已用完」（其链接注册不再发券，其他代理行不受影响）；
    全部代理行用完 → 券码整体「已用完」，并从渠道 / 代理选券等联动下拉中消失（配额语义见 BR-06）。
  • 已停用：运营手动点击停用后，未使用的券码立即过期（带此码链接注册不再发券，但仍可正常注册）；
    已发放到用户手里的券不受影响。
  • 已过期：超出码活动时段自动置「已过期」；
    已领取的券超出已领取券有效期亦自动置为过期（会员侧券状态）。

六、限制条件与校验
  • 手机号白名单仅手机号注册命中（RegType=手机），邮箱注册不命中（针对短信拉新防水军）。
  • 每手机号限领 1 次。
  • 券码停用后立即失效；超活动时段自动过期。
  • 渠道注册码券绑渠道（无代理）；
    代理红包码发给指定子代理（走子代理维度·子代理ID，见 #16，无需手机号白名单）。
  • 适用充值大类必勾至少 1 个。
  • 分配代理批量导入单次最多 5,000 个（逗号 / 空格 / 换行分隔），超出拦截。
  • 白名单导入不符格式（非 区号＋纯数字 10–15 位）的行标错、创建时自动剔除并去重（格式见 3.2）。
