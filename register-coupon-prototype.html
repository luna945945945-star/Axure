PRD · 需求规格说明书
注册拼接充值优惠券系统
一、需求背景与目标
1.1 背景与痛点
  • 渠道 / 代理投放拉新缺少「注册即得券」钩子，发券依赖人工，无法与注册链路自动绑定。
  • 投放数据无法按渠道归因：注册数、首充、二充、送券量散落各处、无统一渠道报表。
  • 短信拉新名单（手机号）在注册环节无法防水军 / 薅羊毛，投放 ROI 难评估。
1.2 需求目标
  • 充值优惠券接入注册链路，支持 4 类适用渠道注册发券。
  • 券码以「券码 × 代理」粒度发放并逐笔可追溯。
  • 建立渠道报表，统一统计注册 / 首充 / 二充 / 充值 / 提现 / 送券。
  • 建立手机号白名单（注册场景号码池）实现圈人 + 转化漏斗追踪，防拉新水军。
二、名词术语
术语定义
充值优惠券码可拼接进注册链接 / 渠道域名的字符码，携带券规格（比例 / 上限 / 门槛 / 已领取券有效期 / 打码量 / 适用充值大类 / 白名单），用户注册命中即发券。
适用渠道券码的发放场景，共 4 类：渠道注册码、人工充值、代理邀请链接、代理红包码。
券码 × 代理粒度同一券码授权给多个代理时，每个代理一行记录，可按代理ID 单独追溯发放进度。
手机号白名单注册场景专用号码池，仅对手机号注册用户命中，用于定向拉新与防水军。
适用充值大类券可用于哪些充值通道大类；充值时按本单大类校验，不符则该券置灰。
券 vs 通道赠金关系用券充值时券赠金与通道赠金的计算关系：互斥·取高 / 叠加 / 互斥·优先券。
三、需求范围（页面清单）
端页面 / 模块类型核心内容
V1后台优惠券管理新增券码列表（券码×代理粒度）+ 创建券码弹窗 + 复制新建 / 停用 / 发放·使用明细
V1后台白名单管理新增注册场景手机号池 + 适用渠道归属 + 导入→注册→充值→用券转化漏斗
V1后台渠道报表改造渠道维度统计（注册 / 首充 / 二充 / 充值 / 提现 / 送券）+ 渠道更新弹窗

四、功能详述
4.1 优惠券管理（V1后台）

券码列表以「券码 × 代理」为一行；顶部支持券码 / 代理ID / 适用渠道 / 状态筛选；「新增券码」打开创建弹窗。

创建券码弹窗字段：

字段必填说明 / 取值
适用渠道是单选：渠道注册码 / 人工充值 / 代理邀请链接 / 代理红包码
券规格是充值比例% + 上限₹（可选不限）+ 最低充值门槛（0=不限，满额才可用券）+ 已领取券有效期（领后 N 天）+ 打码量倍数
适用充值大类是多选·至少勾 1 个，提供「全选」快捷框；充值时按大类校验
券 vs 通道赠金关系是互斥·取高（默认）/ 叠加 / 互斥·优先券
发放限制是每手机号限领 1 次；白名单（渠道注册码 / 代理邀请链接均在此设，属券规格）
分配代理条件仅「代理邀请链接 / 代理红包码」需要，授权哪些代理可推广 / 发放
充值优惠券码自动创建后自动生成，可复制

列表新增字段：适用渠道、上级代理ID、发放进度（点数字看逐笔发放明细：会员 / 手机号 / 时间 / 领取 / 使用状态）、券码使用进度（点数字看逐笔使用明细：会员 / 充值金额 / 用券彩金 / 到账时间）。发放明细、使用明细均支持导出。

列表操作：复制新建——以来源券的规格 / 有效期 / 白名单为模板克隆一张新券，券码重新自动生成、上级代理 / 渠道不继承需重新分配（按券类型自适应），作为独立一条加入列表；停用——立即失效（带此码链接注册不再发券）。

4.2 白名单管理（V1后台）
  • 手机号按「名单 / 批次」归类；顶部漏斗展示 导入 → 已注册 → 已充值 → 已用券 及各级转化率。
  • 新建名单需选「适用渠道归属」（渠道注册 / 代理邀请注册，单选）——标明该名单归属的充值优惠券适用渠道；名单批次下拉与明细列均展示归属渠道。
  • 明细列：所属名单 / 归属渠道 / 手机号 / 上级ID（邀请码归因）/ 状态 / 注册时间 / 充值额 / 用券彩金 / 使用的券。
  • 导入格式校验：手机号 = 国际区号 + 手机号纯数字（10–15 位，如 9188888888666，91=印度区号）；导入 / 导出同格式，粘贴 / 上传时实时校验，不符格式的行标错、创建时自动剔除并去重。
  • 本页只圈人 + 看转化，不在此发券；被哪些券码 / 活动引用会标注。
4.3 渠道报表（V1后台）

渠道维度统计表（支持 日 / 日区间 / 月 / 所有 切换）：渠道名 / 域名 / 编码 / 注册送金·券（编码右侧，彩金 + 充值优惠券合一列）/ 总人数 / 首充人数 / 首充金额 / 二充人数 / 二充金额 / 总充值 / 总提现 / 今日各项 等；行内「渠道更新」弹窗可配置：选择充值优惠券 + 赠送彩金 + 手机号白名单（选券后按券规格只读；不送券·仅彩金时可自定义）。

4.4 资金账变（V1后台 · 改造）

账变类型多选新增「充值优惠券」；充值优惠券到账作为一种账变类型可查，其余字段复用现有账变。

4.5 会员详情（V1后台 · 改造）

「充值提现」区新增充值优惠券奖金总额（仅金额，不含币种 / 张数 = 会员使用充值优惠券获得的奖金累计），并入会员账户全景展示。

4.6 邀请链接（代理端）

代理将「邀请码-券码」拼成推广链接 / 海报拉新；仅推广运营授权（分配代理）给自己的券码。

4.7 带券注册（客户端 · 改造）

注册页解析 invitationCode 中的券码段，按后端判定发券（见「六、状态与判定」）；支持多语言（当前英 / 印地 / 越南，可扩展）。

4.8 数据统计（V1后台 · 改造）

数据统计页「活动」模块下方新增「充值优惠券」统计块，汇总充值优惠券相关数据（发放 / 使用 / 到账金额、人数等）；统计口径与「资金账变」的「充值优惠券」账变一致。

五、业务规则
BR-01
4 类适用渠道：渠道注册码（渠道域名注册送）/ 人工充值 / 代理邀请链接（代理拼链接）/ 代理红包码（发给已注册会员）。
BR-02
分配代理仅「代理邀请链接 / 代理红包码」需要（授权哪些代理可推广 / 发放）；渠道注册码、人工充值无需。
BR-03
手机号白名单归位：白名单属券规格，渠道注册码 / 代理邀请链接均在「建券弹窗」设定；渠道报表引用该券时，白名单按券规格固定为只读，不再在渠道配置单独设。
BR-04
适用充值大类必勾：至少勾 1 个（全选=全部大类可用）；不再是"不勾=全部"。充值页按本单大类校验，不符则该券置灰。
BR-05
券 vs 通道赠金关系：互斥·取高（默认）/ 叠加 / 互斥·优先券，三选一。
BR-06
券码 × 代理粒度：一码多代理 = 多行，可按代理ID 追溯单代理发放进度；可选设「券码发放上限」（0 = 不限，达上限自动置为「已停用」），未设则仅按券码有效期控制。
BR-07
充值优惠券到账逻辑复用现有充值优惠券系统：发出的券进「我的充值优惠券」，充值使用与到账口径与现有充值优惠券系统一致。
六、状态与判定
6.1 券码状态
  • 进行中：活动时段内、未停用，可正常发券。
  • 已停用：运营手动点击停用后，未使用的券码立即过期（带此码链接注册不再发券，但仍可正常注册）。
  • 已过期：手动停用的未使用的券；已领取的券超出已领取券有效期自动置为过期。
6.2 带券注册判定矩阵（后端判定，会员侧无感知）
场景会员侧表现是否发券
券码有效注册成功 + 展示 / 发放充值优惠券（一码一券）发券
券码已过期与无券链接一致，静默普通注册不发
已达发放上限静默普通注册，无感知不发
链接不含券码普通注册不发
七、限制条件与校验
  • 手机号白名单仅手机号注册命中（RegType=手机），邮箱注册不命中（针对短信拉新防水军）。
  • 每手机号限领 1 次。
  • 券码停用后立即失效；超活动时段自动过期。
  • 渠道注册码券绑渠道（无代理）；代理红包码发给已注册会员（走会员维度，无需手机号白名单）。
  • 适用充值大类必勾至少 1 个。
