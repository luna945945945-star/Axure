需求说明：工单类型统计增加商户名称

工单管理 → 工单日报 → 工单类型统计页面，当前列表按「日期 + 商户 + 工单类型」三级维度存储统计数据，但查询结果不返回商户信息。总控账号查看时，同一个工单类型（如「存款未到账」）出现多行重复数据，每行实际是不同商户的统计，但无法区分。

涉及页面
工单管理 → 工单日报 → 工单类型统计
前端改动
总控端：
1. 搜索区新增「商户名称」下拉筛选框（可搜索），支持按商户筛选
2. 列表在「工单类型」列前新增「商户名称」列

商户端：
1. 搜索区「商户名称」显示为只读输入框，自动展示当前登录商户名称，不可修改
2. 列表不显示「商户名称」列（商户端只有自己的数据，无需区分）
后端改动
1. 响应模型 `WorkOrderTypeReportRsp` 新增 `TenantId` 和 `TenantName` 字段
2. `GetTypeReport()` 查询时 Select 增加 `TenantId = p.TenantId`，查出结果后关联 Tenant 缓存取商户名称回填 `TenantName`
3. 请求模型 `WorkOrderTypeReportReq` 新增 `TenantId` 筛选参数（可选），查询增加 `WhereIF`
