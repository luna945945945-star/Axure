1.2 四个核心概念

概念是什么由谁建
集团业务品牌方，运营的最大单位总控端
商户集团下的站点，每个商户独立时区/币种总控端（建时选归属集团）
角色一组菜单 + 按钮权限的打包集团管理员（集团全局）
用户实际登录系统的账号集团管理员

1.3 集团管理员账号特征

✓
你的账号 = 集团管理员
✓ 拥有商户端全部菜单 + 全部按钮权限（无需绑角色）
✓ 顶部商户下拉默认显示一个商户，可切换到本集团任意商户
✓ 可以建立集团下所有角色
✓ 可以建立集团下所有用户
✗ 不能建集团，不能建商户（总控端的事）

1.4 用户类型（3 种）

⚠️
系统没有"账号类型"下拉框，类型由勾选几个商户决定。
用户类型怎么区分谁建典型场景
集团管理员总控端建集团时自动生成总控端整个集团的运营负责人（你）
多商户用户集团管理员建，ManageTenantIds 勾多个集团管理员跨商户的财务/运营/审核
单商户用户集团管理员建，ManageTenantIds 勾 1 个集团管理员商户内具体岗位（客服/出款组员/审核员）

1.5 用户的"职责身份" = 角色决定

账号类型（单/多商户）
  • 单商户
  • 多商户
×
角色（具体职责）
  • 财务主管
  • 出款组长
  • 出款组员
  • 充值审核员
  • 客服
  • 运营

举例：

  • 单商户 + 客服角色 = 某个商户的客服
  • 多商户 + 财务主管角色 = 跨多商户的财务主管
  • 单商户 + 出款组员角色 = 某商户的出款审核员工

1.6 角色作用域：集团全局

集团 A 的角色（集团管理员维护）
├─ 财务主管
├─ 出款组长
├─ 出款组员
├─ 充值审核员
└─ 客服
    ↓ 同一套角色
  商户1 / 商户2 / 商户3 都绑同一套集团角色
核心事实
  • 没有系统内置角色，全部由集团管理员自建
  • 不同集团的角色不互通（每集团独立维护）
  • 同集团内多商户共用一套角色
  • 建角色不需选商户，自动归属当前集团

4.1 出款权限配置（出款审核员必做）

前置依赖：必须先建出款组别

⚠️
派单组别是必填项，必须先建好。
财务管理 → 提现订单管理 → 出款组别配置
↓
【新增组别】
↓
填字段
组别名 + 单员工并发上限 + 待审核超时 + 已审核超时 + 暂停时长上限
↓
保存
组别出现在出款权限的"派单组别"下拉里

配置流程

系统管理 → 系统用户管理，找到该员工 → 点【编辑】
弹窗切到 【出款权限配置】 Tab
填写字段（见下方表格）
点【保存】→ 立即生效，员工下次刷新或登录后能接到符合范围的单

字段说明

字段必填业务影响
派单组别必填该员工归属哪个出款组，继承组的超时规则
金额区间-最小/最大必填仅能审核金额在此区间内的订单
处理上限可选同时持有订单上限；覆盖组别默认值
授权商户可选仅看这些商户的订单；为空 = 用户归属商户范围
币种模式可选法币模式下，同一法币只能选同币种商户
出款权限状态可选关闭 = 无法接单
单笔/每日/每周/每月 提现额度可选该员工可审批的累计金额上限
单日提现笔数可选该员工每天可审批笔数上限

4.2 审核权限配置（含两个独立开关）

🔑
重点：出款审核 vs 人工充值审核是 两个独立开关

审核权限 Tab 结构

① 出款审核权
开关（独立）
控制：能否审核提现订单
② 人工充值审核权
开关（独立）
控制：能否审核人工充值申请
单笔审核额度上限
每日审核额度上限

两个开关的组合策略

出款审核权人工充值审核权适用岗位
开关纯出款审核员
关开纯充值审核员
开开全能财务审核
关关非审核岗位

4.3 人工充值权限配置（三大限额）

💡
给"需要发起人工充值"的员工配（如线下打款补单的财务）。
与 4.2 的"人工充值审核权"是两回事——前者是发起人，后者是审核人。

三大限额

限额项含义
单日发起上限-总次数0 = 不限制 / >0 = 每天最多发起几次
单日金额发起上限0 = 不限制 / >0 = 每天累计金额上限
单笔充值金额上限0 = 不限制 / >0 = 单次金额上限
⚠️
校验规则：达到任一上限 → 再次发起被拒绝 / 清零（0） → 视为不限制（需二次确认）

生效与使用

保存后
↓
财务管理 → 人工充值（发起页）
填：商户 + 会员 + 充值金额 + 充值方式
↓
提交 → 系统校验三大限额
↓
通过 → 进入审核流程
由"人工充值审核权"的员工审

入口：财务管理 → 充值配置 → 充值大类

到手金额 = 充值金额 + 赠送金额 − 手续费

· 百分比赠送：到手 = 充值 + 充值 × 比例
· 固定金额赠送：到手 = 充值 + 固定值

关键字段：自定义名 / 状态 / 最小/最大金额 / 赠送类型 / 赠送值 / 手续费类型 / 手续费值 / 时间窗口 / VIP 等级列表

B

附录 · 配置项速查

B.1 角色

角色名（必填）+ 角色代码（必填，唯一）+ 描述 + 权限树勾选

B.2 用户基础

账号（必填，全局唯一）+ 姓名（必填）+ 密码（必填）+ 角色（多选）+ 归属商户/ManageTenantIds（多选，决定单/多商户）+ 邮箱

B.3 出款权限配置（编辑用户 → 出款权限 Tab）

派单组别（必填）+ 金额区间 [最小,最大]（必填，最大≥最小）+ 处理上限 + 授权商户 + 币种模式 + 出款权限状态 + 单笔/每日/每周/每月 提现额度上限 + 单日提现笔数上限

B.4 审核权限配置（编辑用户 → 审核权限 Tab）

出款审核权（开关）+ 人工充值审核权（开关）+ 单笔审核额度上限 + 每日审核额度上限

B.5 人工充值权限配置（编辑用户 → 人工充值权限 Tab）

单日发起上限-总次数（0=不限）+ 单日金额发起上限（0=不限）+ 单笔充值金额上限（0=不限）

B.6 出款组别

组名（必填，唯一）+ 并发上限[1-100] + 待审核超时[1-9999min] + 已审核超时[1-9999min] + 暂停时长上限[1-9999min]

B.7 提现大类

图标 + 商户 + 系统大类（编辑禁用）+ 自定义名（必填）+ 状态（必填）+ 每日次数(0=不限) + 手续费率(0~1) + 排序 + 最小金额 + 最大金额(≥最小) + USDT 最小/最大

B.8 提现通道

商户（必填）+ 第三方通道映射码（必填）+ 通道名（必填）+ 提现大类（必填）+ 显示名称(≤20，必填) + 商户通道代码(≤32) + 权重(0-9999) + 自动关闭余额 + 状态（必填）+ 每日额度 + 失败重试次数(0-10)

B.9 充值大类

自定义名 + 系统大类 + 状态 + 最小/最大金额 + 赠送类型 + 赠送值 + 手续费类型 + 手续费值 + 时间窗口 + VIP 等级列表 + 商户

B.10 充值通道

状态（三端）+ 金额区间 + 时间窗口 + VIP 等级 + 权重 + 商户绑定 + 自动关闭余额 + 整百开关 + 排序

B.11 通道预警规则

失败率阈值（百分比）+ 余额不足阈值（金额）+ 响应超时（秒）+ 异常次数

B.12 银行映射

第三方银行代码(≤32) + 本地银行代码(≤32) + 银行名（自动）
