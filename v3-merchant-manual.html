07

常见问题

FAQ + 状态枚举速查

7.1 FAQ

▎ 通用 · 用户与权限

Q1. 用户列表为什么有些用户显示"无操作权限"？
A：看用户的归属商户。单商户用户无法编辑多商户用户，所以你是单商户用户时，看到多商户用户会显示"无操作权限"。
📎 详见 模块 1.2 用户类型
Q2. 在用户列表快捷开启「出款 / 人工充值权限」开关，为什么提示"审批商户信息不正确"？
A：点【编辑详情】检查：① 是否配置了权限参数 ② 是否选择了商户。两者缺一不可，快捷开关才能正常启用。
📎 详见 模块 4.3 出款权限 / 4.4 操作
Q3. 职位为主管 / 管理员，为什么无法配置出款权限的参数？
A：设计如此。只有组长 / 组员才配置「指派订单」「商户授权」等参数。主管和管理员不参与审单，无需打卡，系统不向他们派单。
📎 详见 模块 4.3 出款权限
Q4. 删除角色时提示"角色已绑定用户，不能删除"？
A：角色被用户绑定后不能直接删除。两种处理方式：
① 把绑了该角色的用户先解绑（编辑用户 → 改成其他角色 → 保存）
② 或者把这些用户的角色批量转移到新角色上
解绑完成后再来删除该角色。
📎 详见 模块 2 建角色
Q5. 改了角色权限，但已登录用户没立即生效？
A：让用户刷新页面即可同步（不需要退出重登）。如果刷新后仍未生效，让用户退出重登一次。不要让用户保持登录但不刷新，他们看到的是旧的菜单/按钮缓存。
📎 详见 模块 2 建角色
Q6. 新增用户的"权限模板 → 复制现有用户"能复制哪些？密码会复制吗？
A：复制目标用户的：角色 + 归属商户 + 出款权限 + 人工充值权限 全部参数。
不会复制的：账号、密码、谷歌验证。新用户的密码仍需自己输入或选"同上"。
📎 详见 模块 3.1 建用户流程

▎ 出款权限 + 出款组别

Q7. 出款权限的"派单配置组别"下拉是空的？
A：该商户还没建出款组别。先去 财务管理 → 提现订单管理 → 出款组别配置 建好组别（最少 1 个），再回来配出款权限。
📎 详见 模块 4.1 前置：建出款组别
Q8. 转派给我的订单超出了我配的"订单金额区间"，为什么能成立？
A：设计如此。"指派订单配置"只控制系统自动派单时的金额范围；组长 / 主管手动转派给你的订单不受此限制（截图字段下方有 "*转派订单不受此限制" 提示）。
📎 详见 模块 4.3 出款权限
Q9. 修改了出款组别的超时时间（如把"待审核超时"从 10 分改成 5 分），正在处理中的订单会按新值还是旧值？
A：立即按新值重新计算所有在途订单的剩余超时时间。
⚠️ 强烈不建议在大量订单处理高峰期修改，会导致部分订单瞬间超时。建议在低峰期或维护时间窗口操作。
📎 详见 模块 4.1 出款组别配置

▎ 人工充值权限

Q10. "请选择充值权限"的 6 项（人工存款 / 彩金充值 / USDT 充值 / 人工提款 / 彩金扣除 / 退回提现），哪些必须勾？
A：没有"必须勾"的项，按业务岗位决定。常见组合：
· 财务补单员：人工存款 + 人工提款
· USDT 专员：USDT 充值 + 退回提现
· 彩金运营：彩金充值 + 彩金扣除
不勾 = 该用户在前端看不到对应的操作入口。
📎 详见 模块 4.4 操作
Q11. 配置时"是否需要审核 = 关闭"，但角色权限="操作"的用户发起的人工充值还是进入了审核流？
A：这里有双重控制：① 该用户个人的"是否需要审核" + ② 该商户人工充值大类的全局审核要求。任一为"开启"就会进审核。
排查：登录有审核权的账号检查商户级配置；或确认该单是否触发了风控规则强制审核。
📎 详见 模块 4.4 操作

▎ 第三方商户号

Q12. 改了第三方商户号的秘钥后，挂在其下的通道连续失败？
A：按顺序排查：
1. 第三方平台的秘钥是否同步更新（平台和第三方两边要一致）
2. 平台秘钥保存后看 UI 是否显示前 2 字符的掩码（没显示说明保存失败）
3. 用通道行的【测试支付】按钮发一笔小额测试
4. 看通道详情的日志报错信息
建议线上换秘钥前先临时禁用通道，验证通过再启用。
📎 详见 模块 5.2 第三方商户号
Q13. 第三方商户号余额低于"自动关闭阈值"触发后，怎么恢复通道？
A：两步：
1. 通知财务给第三方账户充值至阈值之上
2. 等系统自动同步余额（一般 5-10 分钟），通道自动恢复
如果余额已恢复但通道仍是"关闭"状态，去通道列表手动【启用】该通道（被联动关闭的需要手动开）。
📎 详见 模块 5.2 第三方商户号 / 5.4 充值通道 / 5.6 提现通道

▎ 充值 / 提现配置

Q14. 三端状态都启用了，但用户充值时通道还是不出现？
A：三端只是5 条筛选规则中的 1 条，其他 4 条也要满足：
② 用户充值金额 ∈ 通道金额区间
③ 当前时间 ∈ 通道可用时间段
④ 用户充值等级 ∈ 通道可见等级（如配置了过滤）
⑤ 轮询权重 > 0
任一不满足通道就不会出现。
📎 详见 模块 5.4 充值通道
Q15. 改了提现大类的"提现费率（%）"，已经下单未提交的提现订单按新费率还是旧费率？
A：已下单未提交三方的订单按原费率处理；新发起的订单按新费率。
⚠️ 强烈建议在订单空闲期修改，避免对账困难。修改后通知客服话术变更。
📎 详见 模块 5.5 提现大类
Q16. 提现通道开启"是否整百提现"后，用户输入 150 会怎么样？
A：前端会拦截 + 提示"请输入 100 的倍数"。如果用户绕过前端校验提交，后端接口仍会拒绝。
注意：整百判断只对该通道；如果用户的提现大类下有其他不整百限制的通道，用户会被路由到其他通道。
📎 详见 模块 5.6 提现通道
Q17. 提现通道列表的"实时权重"和我配的"自动出款权重"为什么不一样？
A："自动出款权重"是配置值；"实时权重"是系统按运行情况动态调整后的当前生效值。
系统会根据通道成功率、响应耗时、余额等动态降权或提权。实时权重 = 0 说明系统已临时禁用，看通道详情或日志找原因。
📎 详见 模块 5.6 提现通道

▎ 预警与监控

Q18. 某个通道预警一直在响（不断触发），怎么彻底解决？
A：按预警类型对症处理：
· 失败率超阈值 → 通道有问题，临时禁用并启用备用通道
· 余额不足 → 通知财务补款
· 大额出款 → 业务确认是否正常，正常则【忽略】
· 连续失败 → 排查接口或秘钥
根本问题解决后点【处理预警】标记完成。如果是阈值过低导致误报，去规则配置调高阈值。
📎 详见 模块 6.2 预警规则配置
