0、本期改造范围（一句话说明）

在 V1 现行「按条件全员」老会员回归奖励的基础上，新增「指定 ID 名单」作为另一种"适用对象"。
累充奖励 / 新充值返还的触发条件、金额计算、有效期、领取流程，全部沿用 V1 现行逻辑，本期不动。
本期的唯一逻辑增量：当适用对象 = 指定 ID 时，发放范围限定为名单内会员；其余行为与"按条件全员"一致。

1、列表改造

1.1 顶部筛选

  • 活动状态 下拉（沿用 V1 现有交互：全部 / 已开启 / 已关闭 / 进行中 / 已结束）
  • 适用对象 下拉（全部 / 按条件全员 / 指定 ID）
  • 状态 Tab：进行中 / 已结束（"已结束" Tab 仅出现"指定 ID"规则）

1.2 列变更

  • 表头"ID" → 「活动 ID」
  • 新增列：适用对象（指定 ID 行下方显示 "N 人" 链接进入名单）、累充赠送上限（百分比奖金封顶值，留空 = 不限制）
  • "已结束"表额外列：结束原因（系统自动「发放完成」/ 运营手动）、结束时间

2、新增配置

2.1 适用对象（二选一）

  • 按条件全员（沿用现状）：填写离线天数 / 历史累计充值 / 累计总充值次数 / 累计充提差等过滤条件，凡命中条件的会员都发
  • 指定 ID（本期新增）：跳过条件过滤，直接对名单内会员发放

奖励金额参数（累充固定 / 累充百分比 / 累充赠送上限 / 新充值比例 / 新充值上限 / 有效期）两种适用对象通用，沿用现行参数定义。

2.2 指定 ID 名单维护

数量上限

  • 手动输入 ≤ 500 / 单次导入 ≤ 5000（.xlsx · .csv，提供模板）
  • 单规则累计 ≤ 10000；同站点指定 ID 规则 ≤ 50 条

导入校验结果

  • ✅ 有效 / ⚠ 重复（自动去重）/ ❌ 不存在 / 🚫 已在其他指定 ID 规则中（弹窗确认覆盖）
  • 同 ID 在多条"指定 ID"规则间冲突：取最新规则覆盖，旧规则该 ID 标"被覆盖"

名单冻结

  • 累充开关=开 或 新充值开关=开 任一开启即冻结，禁止追加 / 移除（搜索、导出仍可）
  • 两个开关都关闭即可解冻，恢复增删名单
⚠ 双发风险（系统不去重，运营自查）：同一会员同时命中"按条件全员" + 一条或多条"指定 ID"规则时，奖励独立累加发放（会员领两份甚至多份）。

2.3 开关关闭的处理（关键 · 必读）

🛡 核心原则：开关 ≠ 撤回
关闭累充开关 / 新充值开关 只阻挡"未来新产生"，不影响已经发出去的奖励。已发未领的钱按既定生命周期让会员领走，不做事后回收。
理由：① 用户已收到站内信红点 / 推送通知，事后作废必然客诉；② V1 是 H5 套壳，没有反向 push 召回手段；③ 财务侧"已发 = 已计提"，回收会产生对账噪声。

业务行为表 + QA 用例（一张表搞定）

场景开关操作预期行为
会员 A 累充奖励已发未领关闭累充开关✅ A 登录仍可领取，按发放时金额到账
会员 B 在名单内但未达成累充条件关闭累充开关❌ B 登录不发放
会员 C 在新充值返还有效期内关闭新充值开关✅ C 在原有效期内继续充值仍按既定比例返还
会员 D 尚未触发新充值返还关闭新充值开关❌ D 之后充值不再返还
会员 E 同时有累充已发未领 + 新充值在有效期同时关闭两个开关✅ 两笔都能正常完成（已发未领不撤回）

2.4 规则生命周期（按"适用对象"分两套）

① 按条件全员：只有「进行中」一种状态。新会员持续达标、持续发放，没有"消费完"概念，因此不会进入"已结束"。
② 指定 ID：名单有限，有明确的"发完"时间点，因此存在「进行中 → 已结束」流转。
两种规则均不设自动归档 / 软删除；如需清理由运营手动删除。

① 按条件全员

  • 仅一种状态 = 进行中
  • 两个开关都关闭 = 暂停生效，但规则仍在「进行中」Tab，运营可随时重启
  • 开关层面的"开启 / 关闭"通过列表顶部 「活动状态」筛选器 定位（沿用 V1 现有交互），不另开 Tab
  • 需废弃 → 运营手动「删除」（二次确认）

② 指定 ID

流转判定条件
进行中 → 已结束 系统判定"两条线都不会再产生新发放"：
· 累充侧已终止（开关=关 或 名单内所有应发会员均已发完）
· AND 新充值侧已终止（开关=关 或 全员有效期到期 或 全员累计达上限）
注："终止"仅指不再新发，不撤回存量已发未领（参见 §2.3）
已结束 → 删除运营手动删除，已发记录保留供财务对账
  • 已结束 = 终态，业务上不会恢复（复用场景请用「复制」生成新规则）
  • 已结束后存量"已发未领"和"待返还"奖励仍可正常领取 / 派发
  • 领取与否在「回归奖励记录」页按 活动 ID + 状态 筛选查询，与规则状态解耦

2.5 复制规则

  • 已结束规则 → 「复制」生成新规则，预填配置
  • ID 名单选项：不复制 / 全部复制 / 仅复制未触发累充的
  • 新规则需记录"复制自哪条规则"以便追溯

3、回归奖励记录改造

  • 筛选新增：适用对象（按条件全员 / 指定 ID）、活动 ID
  • 列新增：适用对象、活动 ID（可点击跳到规则配置）
  • 每条记录需可反查"来自哪条规则" + "属于哪种适用对象"，供运营 / 财务排查
  • 规则被删除后已发记录保留（财务一致性）
