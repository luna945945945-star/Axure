1、新增字段

1.1 搜索框

  • 适用对象 下拉筛选
  • 状态切换 Tab：进行中 / 已结束 / 已归档

1.2 列表新增列

  • 状态、适用对象（指定 ID 行下方显示"N 人"链接进入名单）
  • 已结束 额外列：结束原因（自然结束 / 运营手动）、结束时间、距归档倒计时
  • 已归档 额外列：结束时间、归档时间、距软删倒计时

2、新增配置

2.1 数量上限

  • 手动输入 ≤ 500
  • 单次导入 ≤ 5000（.xlsx / .csv，提供模板）
  • 单规则累计 ≤ 10000
  • 同站点指定 ID 规则 ≤ 50 条

2.2 累充奖励发放

  • 触发：保存 + 累充开关=开 → 异步队列遍历名单立即结算
  • 计算：固定奖金 + 历史累充 × 百分比
  • 频率：每会员每规则仅一次；跨规则站点级生命周期最多一次
  • 已发不撤回

2.3 新充值返还

  • 触发：名单内会员有效期内每次新充值
  • 计时起点：该会员累充奖励到账时刻（每个 ID 独立计时）
  • 频率：有效期内多次，受单笔上限约束
  • 失效：到期 / 累计达上限 / 新充值开关关闭 / 会员被移除

2.4 名单冻结

  • 条件：累充开关=开 或 新充值开关=开（任一开启即冻结）
  • 限制：禁止追加、禁止移除（搜索、导出仍可）
  • 解冻：两个开关都关闭

2.5 ID 冲突与双发风险

同一会员同时命中"全员" + "指定 ID" → 奖励独立累加发放（领两份）。系统不去重，运营自查。
  • 同 ID 在多条"指定 ID"规则间冲突 → 取最新规则覆盖，旧规则该 ID 标"被覆盖"
  • 导入校验：✅ 有效 / ⚠ 重复（自动去重）/ ❌ 不存在 / 🚫 已在其他规则（弹窗确认覆盖）

2.6 规则生命周期

流转触发条件
进行中 → 已结束累充全发完 AND（新充值有效期到期 OR 开关关闭），每小时定时扫描
已结束 → 已归档距实际结束时间满 AutoArchiveDays（默认 30 天）
已归档 → 软删除距归档时间满 180 天，UI 不可见数据保留
  • 已结束：开关 disabled，名单只读
  • 已归档：完全只读，不可删除

2.7 复制规则

  • 已结束 / 已归档 规则 → 「复制」生成新规则，预填配置
  • ID 名单选项：不复制 / 全部复制 / 仅复制未触发累充的
  • 新规则记录 SourceRuleId 字段

3、回归奖励记录

  • 筛选新增：来源类型（系统自动 / 指定 ID）、规则 ID
  • 列新增：来源类型、规则 ID（可点击跳到规则）
  • 字段：每条记录新增 RuleId + SourceType
  • 删除规则后已发记录保留（财务一致性）
