2新增配置说明

2.1 适用对象（二选一）

类型说明
按条件全员（沿用现状）填写离线天数 / 历史累计充值 / 累计总充值次数 / 累计充提差等过滤条件，凡命中条件的会员都发。
指定 ID（本期新增）跳过条件过滤，直接对名单内会员发放。
奖励金额参数累充固定 / 累充百分比 / 累充赠送上限 / 新充值比例 / 新充值上限 / 有效期 — 两种适用对象通用，沿用现行参数定义。

2.2 名单维护（仅指定 ID 规则）

项说明
数量上限手动输入 ≤ 500；单次导入 ≤ 5000（.xlsx / .csv）；单规则累计 ≤ 10000；同站点指定 ID 规则 ≤ 50 条。
导入校验有效 / 重复（自动去重）/ ID 不存在 / 已在其他指定 ID 规则（弹窗确认覆盖）。冲突取最新规则，旧规则标"被覆盖"。
新建确认后两个开关默认都未开启 → 名单可自由编辑（追加 / 移除 / 重新导入）。
开启任一开关累充开关 或 新充值开关 任一开启 → 名单立即冻结（禁止追加 / 移除，仅可搜索 / 导出）。
再次编辑名单必须先关闭两个开关 → 编辑名单 → 重新打开开关。
关开关期间存量"已发未领"不受影响，照常领取 / 派发。

2.3 双发风险与约束

维度说明
同规则 · 累充奖励唯一性键 = 活动 ID + 会员 ID。同一规则同一会员最多发一次累充。重新打开开关扫描名单时，已发过的会员自动跳过。
同规则 · 新充值返还唯一性键 = 活动 ID + 会员 ID。同一规则同一会员最多开启一次返还有效期，已开启过的不重置；已过期的不重新开启。
跨规则双发风险系统不去重，运营自查。同一会员同时命中"按条件全员" + 一条或多条"指定 ID"规则时，奖励独立累加发放（会员领两份甚至多份）。导入名单前请与现有全员规则做交集排查。

2.4 规则生命周期

类型说明
按条件全员仅"进行中"状态。新会员持续达标、持续发放，无"消费完"概念。两个开关都关闭 = 暂停，规则仍在「进行中」Tab。开关层面"已开启 / 已关闭"通过顶部「活动状态」筛选器定位。
指定 ID · 自动结束累充全部发完 AND 新充值有效期到期 → 系统判定，自动置为「已结束」。
指定 ID · 手动结束列表行的「结束」按钮 → 二次确认，置为「已结束」。
已结束 = 终态存量"已发未领"仍可正常领取。新场景请新建规则（不提供复制功能）。

3开关逻辑 必读

🛡 核心原则：开关 ≠ 撤回。关闭累充开关 / 新充值开关 只阻挡"未来新产生"，不影响已经发出去的奖励。已发未领的钱按既定生命周期让会员领走，不做事后回收。

场景开关操作预期行为
会员 A 累充奖励已发未领关闭累充开关✅ A 登录仍可领取，按发放时金额到账
会员 B 在名单内但未达成累充条件关闭累充开关❌ B 登录不发放
会员 C 在新充值返还有效期内关闭新充值开关✅ C 在原有效期内继续充值仍按既定比例返还
会员 D 尚未触发新充值返还关闭新充值开关❌ D 之后充值不再返还
会员 E 同时有累充已发未领 + 新充值在有效期同时关闭两个开关✅ 两笔都能正常完成（已发未领不撤回）
