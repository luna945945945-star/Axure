📋 需求摘要：V3 后台 ① 「提现订单管理」列表页"提现账户详细"列 + ② 「审核订单详情」弹窗对应区域，两处都新增"变更提示"。按本次提现大类智能匹配检查范围（V3 当前 9 大类：BANK CARD / UPI / USDT / PIX / Easypaisa / JAZZCASH / E-Wallet / KBZPay / WavePay）：
  • 🔴 有变更 → 红色提示"用户提现 {大类} 信息变更，请核查"
  • 🔵 首次该大类提现 → 蓝色提示"用户首次 {大类} 提现，请核查 {账户} 信息"
  • ⬜ 无变更 + 非首次 → 不显示任何提示
展开后仅显示修改日期，不展示修改内容—— 由审核人员自行核实。

📌 业务规则

规则说明
大类定义沿用 V3 现有 9 大类：BANK CARD / UPI / USDT / PIX / Easypaisa / JAZZCASH / E-Wallet / KBZPay / WavePay
检查表对应关系每个大类对应各自的提现账户表（如 BANK CARD → tab_WithdrawalBankCard；USDT → tab_UsdtAddress；UPI / PIX / 其他 → 各自账户表），具体表名以开发确认为准
判定时间窗（基准）该用户同大类最后一笔 Status = PASS（已通过）的订单完成时间 → 本次提现申请时间。撤销 / 拒绝 / 超时订单不算基准
首次大类提现从未有过该大类 PASS 订单 → 基准用该大类首次绑定时间。提示文案换成"🆕 用户首次 {大类} 提现，请核查 {账户} 信息"
同大类范围用户名下所有该大类账户（不只是本次使用的那一个）的 Add / Update / Delete 全部纳入。理由：新增账户、删除旧账户都是资金转移风险信号
判定字段对应账户表的所有业务字段（如 BANK CARD：银行名称/收款人/账号/IFSC ｜ USDT：钱包地址/链/备注 ｜ UPI：UPI ID 等）。任一字段在窗内被新增/修改/删除即触发
展示位置① 列表「提现账户详细」列底部；② 审核订单详情弹窗对应账户区域底部
展示形式默认收起，仅一行提示 + ▼；点击展开显示最近修改日期，不展示具体变更内容
颜色规则（仅两色）🔴 红 = 有变更（任何大类统一红色）· 🔵 蓝 = 首次该大类提现
为何不展示修改内容由审核人员自行核实，避免界面冗余；展开后只给一个时间锚点
显示条件仅待审核订单显示（已通过/已拒绝订单不显示，减少干扰）
是否拦截审核不拦截，仅展示提示，由运营自行判断是否通过
