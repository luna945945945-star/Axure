💳 充值通道手续费展示改造需求原型 v1

当三方厂商通道收取手续费、后端下发手续费字段时，客户端需展示手续费并新增手续费说明；后台 5 个位置同步新增手续费字段。适用系统：V1（H5 套壳）+ V1 后台

一、需求描述

部分三方厂商充值通道会对充值收取手续费（如越南、马来等充值通道）。目前手续费只在后端用于与三方结算，会员充值时看不到、运营在后台记录里也查不到：会员不清楚"充 ₹100 实际到账多少"，容易产生客诉；财务对账时缺少手续费维度，难以与三方结算单核对。

需求来源：运营 / 财务反馈 ＋ 新接入高手续费通道的上线需求。
需求目标：让手续费在会员端透明可见（充多少、实际上分多少），并在后台通道配置与充值记录中可配置、可查询、可对账。

三、需求说明（业务规则）

  1. 手续费配置：手续费比例在充值通道上配置，每个通道独立配置；填 0 表示该通道不收取手续费。
  2. 展示条件：仅当通道手续费 > 0 时，会员端才展示手续费及手续费说明；手续费 = 0 的通道页面与现状完全一致，会员无感。
  3. 到账口径：手续费从会员到账中扣除——会员实付 ＝ 充值金额，实际上分 ＝ 充值金额 − 手续费；有优惠时 实际上分 ＝ 充值金额 − 手续费 ＋ 充值奖励。
  4. 场景一 · 有手续费无优惠：通道展示"手续费 X%"，金额下方展示"实际上分"，充值说明区新增一条手续费文案。
  5. 场景二 · 无手续费：通道不展示手续费标签，实际上分 ＝ 充值金额，说明区提示"不收手续费·全额到账"，不出现手续费文案。
  6. 场景三 · 有奖励＋手续费：金额下方展开明细公式（充值金额 ＋ 充值奖励 − 手续费 ＝ 实际上分），奖励为正、手续费为负；多种奖励（充值奖励、首充奖励…）各占一行。
  7. 充值记录展示：会员端"充值记录"里，若订单扣了手续费，展示 金额（充值金额）/ 手续费 / 实际上分 三项，让会员清楚"充了多少、实际到账多少"；未扣手续费的订单保持原样，不新增行。
  8. 实时联动：会员切换通道、修改充值金额或选择快捷金额时，手续费与实际上分实时刷新。
  9. 多语言：手续费、实际上分、手续费说明等文案随会员语言切换（中文 / 英语 / 印地语 / 越南语…），金额数值与货币符号格式保持一致。
  10. 后台配置与记录：运营可在通道"常用修改"配置手续费率并在通道列表查看；三方待入款、入款记录展示每笔订单手续费（取下单时快照，通道费率后续调整不影响历史订单），供财务与三方结算对账。
  11. 边界与精度：手续费导致到账出现小数时按币种精度处理（如 INR 保留 2 位小数），保证"充值金额 − 手续费 ＝ 实际上分"可对账；旧通道无手续费字段时默认视为 0，保持现状。

四、关联影响（后端需同步改造）

口径已定：会员到账与下游统计 / 返佣 / 打码 / 活动阈值，一律以「实际上分（＝ 充值金额 − 手续费）」为准。引入手续费后「充值金额」与「实际上分」出现差额，以下模块（V 系统现多按充值金额计算）需按此口径同步改造：
模块改造点优先级
财务报表 / 充值统计充值日报·周报、充值总额、日/周奖励统计改按「实际上分」汇总（V1 部分现按充值金额，需一并调整）高
代理返佣 / 返水 / 分红返佣基数改按「实际上分」，避免引入手续费后多付返佣成本高
打码量 / 流水稽核 / 提现条件「产出与充值比」等稽核的充值基数改按「实际上分」高
充值活动 / 首充 / 累计充值 / VIP达标阈值（累计充值满 X、首充满 X 等）改按「实际上分」判定中
资金流水 / 上分对账上分金额以「实际上分」入账（V2 中台已如此，V1 统一）中
历史充值汇总表会员历史充值总额统计表（AChartRecharge 等）口径统一为「实际上分」；历史数据是否回溯另行评估低
口径统一原则：会员到账、财务统计、代理返佣、打码稽核、活动阈值，全部以「实际上分（＝ 充值金额 − 手续费）」为口径；手续费按币种精度保留 2 位小数（如 INR），保证"充值金额 − 手续费 ＝ 实际上分"可对账。

🧭 三种场景对照

  • ① 有手续费：通道显示「手续费 X%」，金额下方「实际上分 = 充值 − 手续费」，说明区一条手续费文案
  • ② 无手续费无优惠：与现状完全一致——不展示"实际上分"，也不加手续费说明，保持原样
  • ③ 奖励+手续费：金额下方展开明细卡「充值 + 奖励 − 手续费 = 实际上分」，奖励绿色、手续费红色

🌐 多语言

  • 手续费 / 实际上分 / 手续费说明 等文案随语言切换：中文 · English · हिन्दी(印地语) · Tiếng Việt(越南语)
  • 货币符号 ₹、金额格式、数字在各语言下保持一致
  • 点顶部语言按钮，三个场景整体切换
