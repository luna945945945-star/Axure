2.3 站内信 与 推送级别（两个开关解耦）

站内信（默认开启）= 写消息中心红点，所有活动达标都会写一条站内信记录 + 顶部铃铛红点。
是否推送 + 推送级别 = 在站内信之上叠加的实时弹窗（L2 横幅 / L3 强弹窗），关闭后仅保留站内信红点。
级别表现默认匹配
L2 横幅 toast顶部下滑 3-5 秒活动类（9/10）
L3 强弹窗居中模态 + 遮罩，需用户主动关钱类（2/3/4）+ KYC（11）

原 L1 静默红点 已不再作为推送级别选项，"仅红点"的能力由"站内信"独立开关承担，便于运营单独控制是否写消息中心。

2.4 编辑 / 保存约束

  • 表格直接编辑（不进入编辑态）→ 改动行打脏标识 → 顶部「💾 保存(N)」常亮
  • 关闭/跳页拦截：有未保存改动时浏览器拦截
  • 批量动作仅改"是否推送"，跳转动作 + 推送级别保留（避免覆盖运营精配）

3、推送链路与兜底

3.1 触发链路

业务达标 ↓ 判断「站内信」开关 → 开 → 写 tab_SiteMessage（顶部铃铛红点） 关 → 不写消息中心 ↓ 判断「是否推送」开关 ↓ 开 SignalR 推送在线端 → 按级别渲染 L2 横幅 / L3 强弹窗 ↓ 离线 仅依赖站内信红点 → 登录时拉取消息中心

3.2 跳转映射

配置行为
不跳转仅关闭横幅 / 红点
活动页跳到该活动详情页（按 ACT_CONFIG_PAGES 映射）
钱包 / 活动页 / 提现页 / VIP 中心跳对应页
