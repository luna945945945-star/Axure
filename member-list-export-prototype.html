一、新增筛选项（5 项）

1充值总次数 ≥ N 实时
定义：按会员累计成功充值笔数筛选（全时段累计，不随日期/快捷日期变化）。
规范/限制：入参整数 N≥0；填 0 特指"未充值(=0)"会员，填 N（≥1）按"充值笔数 ≥ N"处理；只统计充值成功（已到账）的订单，不含赠送/后台加币/红包等非充值入账；空值不参与筛选。
2未登录天数 ≥ N 实时
定义：筛出"距最后一次在线/活跃已达 N 天及以上"的会员；天数 = 今天 − 最后在线日期（今天在线过 = 0 天）。
规范/限制：入参正整数（N≥1）；从未在线过的会员视为"未登录天数 = ∞"，满足任意 ≥N；查询按"最后在线日期 ≤ 今天−N"处理走索引；口径统一取"最后在线/活跃时间"，与列表"最后在线"列同源。
3流水倍数 ≥ N T+1
定义：筛出"流水倍数 ≥ N"的会员；倍数 = 有效投注 ÷ 总充值（均为全时段累计）。
规范/限制：入参支持小数（如 1.5）；无充值（分母为 0）的会员不计入、列表显示"—"（如需排查有投注、零充值的会员，请改用"充值总次数 = 0 且 总流水 > 0"筛选）；比较用精确值（非显示舍入值）；分子分母均取 T+1 口径。
4总盈亏 T+1
定义：操作符（≥ / ≤）可切换 + 填数值：≥N 筛盈亏不低于 N（找盈利/巨赢，如 ≥1000）；≤N 筛盈亏不高于 N（找亏损，填负数，如 ≤ -1000 = 亏损达 1000 及以上）。
规范/限制：入参支持负数与小数；正数=会员盈利、负数=会员亏损（会员视角）；口径为"真实输赢"、按有效投注、T+1 截止昨天。⚠️ 约定正数 = 会员盈利，开发须核对统计字段正负方向与此一致（平台视角相反需取负）。
5银行卡绑定手机号（输入手机号查询）实时
定义：输入手机号，精确查询绑定该手机号的提现银行卡对应会员。
规范/限制：按会员绑定提现银行卡时预留的手机号匹配；一人多卡时任一卡的绑定手机号命中即返回；已解绑的卡不参与匹配（按当前有效状态）；只匹配银行卡类型（排除钱包/UPI 等其他提现方式）；与列表"银行卡绑定手机号"显示同口径；无查看手机号权限的账号按脱敏规则处理。

二、新增「自定义列」功能

入口：列表工具栏、快捷日期条（今天/昨天…）左侧的"⚙ 自定义列"按钮，实时显示"已显示 X/N"。
  • 勾选显隐：竖向列表逐项勾选，勾掉即从列表移除，即时生效。
  • 拖拽排序：拖左侧手柄调整列先后顺序。
  • 星标置顶：点 ☆ 将该列固定排到最前。
  • 重置：一键恢复默认列顺序与显隐。
  • 会员ID 必显锁定，不可取消勾选。
  • 偏好本地保存：存浏览器本地（localStorage），下次打开自动套用；纯前端、不改后端、换设备不同步。
⚙️ 纯前端视图层，不改后端查询，约 1 人日。
⚠️ 若列表底部有"合计/总计"行，金额合计格须随对应列一起隐藏，避免错位。

三、新增列表字段（5 列）

1银行卡绑定手机号 实时
定义：会员绑定提现银行卡时预留的手机号。
规范/限制：绑定多张时默认只显示主卡，行内 ▾N张 点开按绑定时间倒序列出其余（默认收起）；主卡 = 最近一次绑定的银行卡；已解绑的卡不显示；只取银行卡类型；按权限脱敏。
2未登录天数 实时
定义：会员距最后一次在线/活跃已过去的天数（= 今天 − 最后在线日期）。
规范/限制：与筛选项 2 同源（最后在线/活跃时间），与列表"最后在线"列一致；从未在线的会员显示"未登录"（筛选中视为满足任意 ≥N）；今天在线过显示 0。
3总流水（有效投注）T+1
定义：会员累计有效投注金额（不含撤单、无效局等），全时段累计。
规范/限制：统计到昨天为止；无投注会员按 0 显示；金额保留 2 位小数、千分位。
4总盈亏 T+1
定义：会员累计输赢，正数盈利、负数亏损（会员视角），按有效投注计算。
规范/限制：统计到昨天为止；正绿负红，=0（含无投注）显示中性色；正负方向与筛选项 4 一致。
5流水倍数 T+1
定义：有效投注 ÷ 总充值，衡量会员投注活跃度。
规范/限制：无充值（分母 0）显示"—"；保留 1 位小数，筛选比较用精确值；T+1，与筛选项 3 同口径。

四、导出说明

  • 导出当前筛选条件下的全量数据，与列表共用同一套查询条件，新增筛选/字段对导出自动生效。
  • 格式 CSV；导出字段含全部新增列。
  • 手机号 / 银行卡类字段按操作人权限自动脱敏（无权限时带 * 脱敏，非置空）。
  • 多张银行卡绑定手机号：导出无展开/收起，同一单元格内用「；」拼接全部卡号（每个仍按权限脱敏），主卡排首位；不拆多列、不拆多行。
  • T+1 字段（总流水/总盈亏/流水倍数）导出同为"截止昨天"口径。
  • 单次导出上限 20 万条；接近上限时走流式写盘，避免内存压力。

五、权限控制

  • 复用现有按钮级权限体系控制敏感字段的查看与导出。
  • 查看手机号：无权限时列表打星脱敏；导出手机号：无权限时导出同样带 * 脱敏（与列表打星同规则，非置空）。
  • 银行卡绑定手机号沿用同一套脱敏规则（新增权限点或复用现有手机号权限，待定）。
  • 批量操作按钮（拉黑/禁用/转代理等）权限沿用现状，本期不改。
