三、功能详述
3.1 人工充值发券（V1后台 · 改造）

路径：财务管理 → 人工充值。存款类型新增「充值优惠券」；非券类型仍为 存款金额 + 打码量设置、流程不变。以下按页面从上到下分模块：① 顶部切换 → ② 会员查询 → ③ 发券表单 → ④ 批量导入弹窗。

① 顶部切换（单个会员 / 批量导入）

控件类型取值 / 说明
单个会员分段 tab（默认）逐笔发放：查询会员 → 配券 → 提交（见 ②③）
批量导入分段 tab点击打开「导入界面」弹窗，按名单批量发放（见 ④）

② 会员查询区（单个会员）

控件 / 字段说明 / 取值
会员ID + 查询输入会员ID 点「查询」，命中后方可发放；未命中不展示信息、不可提交
会员账户 / 用户昵称命中回显供核对（如 9188****8666 / test002）
会员ID / 手机·邮箱命中回显
账户余额 / 打码量命中回显（余额高亮）
当日充值 / 当日彩金命中回显，辅助风控核对

③ 发券表单（存款类型选「充值优惠券」时隐藏 存款金额 / 打码量、显示券规格 + 发放方式，见 BR-01）

字段必填说明 / 取值
存款类型是人工存款（加存款）/ 人工提款（减存款）/ 彩金充值（加彩金）/ USDT充值 / 充值优惠券（新）；
选券类型后展示以下券规格 + 发放方式
券来源是仅券类型出现：从券码库选 / 现填规格（快捷）；券码库＝唯一券源、仅列「适用人工充值」的券且规格只读；现填的也自动登记成券码（BR-05）
选择券码条件券来源＝券码库时出现：下拉仅列适用人工充值券（如 MANUAL8 · 充值8% / 上限₹40 · 15天），规格只读带出
优惠比例是现填规格：%
金额上限是现填规格：封顶 ₹N / 不限，二选一
已领取券有效期是现填规格：天
最低充值门槛是现填规格：₹，0＝不限；单笔充值 ≥ 此额才可用券
通道赠金关系是现填规格：互斥·取高（默认）/ 叠加（券+通道都给）/ 互斥·优先券；计算口径见 3.4
适用充值大类是现填规格：多选 UPI / 银行卡 / USDT / E-Wallet / ARPay（含「全选」），至少勾 1 个
券使用限制否现填规格：勾「仅首充有效」后仅用户首次充值可用，非首充置灰
打码量设置是现填规格：倍（彩金按打码倍数计入流水）
发放方式是仅券类型有：自动领取 / 弹窗领取——两者均发放即入袋，弹窗仅多一个到账告知弹窗、券均已入账（BR-02）
预览自动券面 + 充 ₹100 / 500 / 1000 试算（未达门槛显示「未达门槛」）+ 通道赠金关系
备注否操作备注，留存于操作 / 审核记录
充值 / 发放按钮提交；是否进审核流按操作人账号权限自动判定（BR-04），金额 / 数量不作判定

④ 批量导入弹窗（顶部「批量导入」打开「导入界面」）

区块 / 控件说明 / 取值
导入类型（新）单选：充值优惠券 / 其他类型（人工充值 / 彩金充值 / USDT / 彩金扣除），单次仅限一种
模板下载 / 上传Excel下载对应模板 → 上传；单次 ≤ 1.5 万条、文件 ≤ 4M（xlsx / csv）
充值优惠券模板列账户类型 / 会员ID / 充值类型 / 备注；无充值金额列、无打码量列——打码量随券规格走，在券配置区统一配
其他类型模板列账户类型 / 会员ID / 充值金额 / 充值类型 / 打码量倍数 / 备注；金额 / 打码量按导入列执行
校验结果（三态）逐行展示校验信息 + 状态，顶部汇总「校验错误 / 重复提醒 / 总数」：
失败（已持未使用同类券）· 提醒（会员ID 重复——不自动去重，标出重复项告知运营，由运营自行决定是否删除；同一会员可发多张，属正常场景）· 待执行（校验通过）；
仅「待执行」进入发放 / 审核（提醒项经运营确认保留后转「待执行」）
券配置区（充值优惠券）券来源（券码库 / 现填规格）+ 券规格（同 ③）+ 通道赠金关系 + 券使用限制 + 适用充值大类 + 发放方式；全名单统一，名单只管发给谁（会员ID），对名单内每人各发一张同规格券
其他类型配置人工充值 / 彩金充值 / USDT 均直接到账、彩金扣除为扣款，无发放方式（BR-03）
底部执行按操作人权限（BR-04）：免审 →「开始执行」直接发放；需审核 →「提交审核」入发券审核

3.4 赠金计算规则（券 vs 通道赠金）

本节是「券赠金」与「通道赠金」的唯一口径，#31 / #16 / #20 三份需求共用；其余章节与业务规则只引用本节、不重复。

业务流程：用户用券充值时，实际到账 = 实付金额 ＋ 赠金 − 手续费。「赠金」由两部分竞争或叠加得到——券赠金（这张券带来的）与通道赠金（充值方式自带的活动奖励）；
二者谁生效、是否相加，由券规格里配置的赠金关系决定（三选一，运营建券时设定，代理 / 会员不可改）。

字段定义：

概念说明
实付金额用户本次实际充值金额；
券门槛、券比例、手续费均以它为基数（不以到账额为基数）
券赠金实付金额按券比例计算，不超过券上限；
未达券门槛则该券不可用
通道赠金充值方式自带的活动奖励，含三类：充值通道赠金 / VIP等级充值奖励 / 快捷金额充值奖励
手续费按实付金额的固定费率扣取
打码量赠金到账后、提现前需完成的流水倍数；
券赠金与通道赠金各自按自己的倍数计算
取整规则券赠金、通道赠金、手续费均向下取整为整数；
展示统一保留 2 位小数（如 50 显示为 50.00），各自单独计算

三种赠金关系：

赠金关系含义券是否消耗到账明细
互斥·取高
（默认）		券赠金与三类通道奖励中只发金额最高的一个；
金额相等时优先发券		券胜出 → 消耗；
券未胜出 → 不消耗、退回券包，不计入「已使用」		一行「奖励」＋来源标注
叠加券赠金与三类通道奖励全部发放、金额相加消耗两行：「充值奖励」（三类通道合计）＋「充值优惠券彩金」；
两行金额相加 = 总赠金
互斥·优先券无条件只发券赠金，不与通道比较（即使通道更高）消耗一行「奖励」＋来源标注

来源标注：明细为一行时（取高 / 优先券）必须标注赠金来源，让会员看清钱从哪来、券有没有被扣：

  • 券胜出：奖励 +₹50（充值优惠券 10%，高于通道 ₹45）
  • 券未胜出：奖励 +₹90（VIP等级奖励 高于券 ₹50；券未使用、已退回）

算例（券 10%、上限 ₹50、无门槛；通道奖励 9%；手续费 2%）：

实付金额券赠金通道赠金取高·赠金叠加·赠金优先券·赠金
₹500₹50₹45₹50（券胜出）₹95₹50
₹1000₹50（触上限）₹90₹90（通道胜出，券退回）₹140₹50
₹333₹33₹29₹33₹62₹33

到账 = 实付金额 ＋ 赠金 − 手续费（手续费 = 实付金额 × 2%）。
例：充 ₹500 取高，到账 500 ＋ 50 − 10 = ₹540；
充 ₹1000 取高，到账 1000 ＋ 90 − 20 = ₹1070。可作验收基准。

四、业务规则
BR-01
存款类型选「充值优惠券」：隐藏 存款金额 / 打码量，显示 券规格 + 发放方式。
BR-02
发放方式仅「充值优惠券」有：
自动领取＝券直接入袋 + 站内信通知；
弹窗领取＝券同样发放即入袋，只是会员登录首页时多弹一个到账告知弹窗——弹窗仅为通知，点不点「领取」、有没有看到，券都已在账户里；
两种方式券归属逻辑相同，均无「领取截止」、无未领作废。
BR-03
彩金充值 / 人工充值 / USDT 直接到账、彩金扣除为扣款 → 均无发放方式（非券的资金操作没有"领取"概念）。
BR-04
是否审核沿用现有「人工充值」审核逻辑：由操作人后台账号的权限决定、系统自动判断（不在发券表单手动勾选）——需审核账号提交的发券（单个 / 批量一视同仁）一律进「发券审核」待复核，免审账号直接执行；
金额与数量均不作为判定条件；
权限与「是否需要审核」开关的配置来源见 3.2 末「后台操作权限配置」。
BR-05
券来源：券码库＝唯一券源；现填规格也自动登记成券码。批量时名单只管发给谁（会员ID），券规格 / 发放方式在弹窗统一配置。
BR-06
券 vs 通道赠金关系：互斥·取高（默认）/ 叠加 / 互斥·优先券，三选一（属券规格 —— 从券码库选时随券只读带出，现填规格时录入）。计算口径见「3.4 赠金计算规则（券 vs 通道赠金）」，本条不另立算式。
五、限制条件与校验
  • 批量单次 ≤ 1.5 万条，文件 ≤ 4M（xlsx / csv）。
  • 校验：已持未使用同类券（同规格 / 同类型） → 失败；
    会员ID 重复 → 提醒运营（不自动去重，标出重复项，由运营自行决定是否删除——同一会员可发多张券属正常场景）；
    单次仅限同一充值类型。单会员发放同样校验已持同类券。
    注：充值优惠券不做黑名单校验。
  • 现填规格须做合法区间校验（比例% / 上限₹ / 已领取券有效期 / 最低充值门槛 / 打码量倍数；适用充值大类至少勾 1 个），避免录入异常规格。
  • 批量失败 / 忽略清单沿用现有「批量充值」的失败处理逻辑（逐行跳过 + 单条隔离 + 结果清单带状态与原因，复用现有流程；不含失败清单导出 / 修正后重传）。
  • 口径：会员主动获取（红包码兑换 / 注册得券）一律自动入袋；仅充值优惠券定向发可选弹窗领取。
