三、功能详述
3.1 人工充值发券（V1后台 · 改造）

单个会员表单字段：

字段必填说明 / 取值
存款类型是人工存款 / 人工提款 / 彩金充值 / USDT / 充值优惠券（新）
券来源是选充值优惠券时出现：从券码库选 / 现填规格；券码库仅列「适用渠道 = 人工充值」的券，规格只读
券规格是比例 / 上限 / 已领取券有效期 / 打码量；券码库来源为只读
发放方式是仅充值优惠券有：自动领取 / 弹窗领取（弹窗须设领取截止）

批量导入弹窗：导入类型（充值优惠券 / 其他类型[人工充值 / 彩金充值 / USDT / 彩金扣除]）、模板随类型切换（券模板无充值金额列）、券配置（券来源 + 发放方式，仅列适用人工充值的券）、上传 → 校验 → 执行。

批量校验三态：上传后逐行校验并给结果 —— 失败（黑名单会员 / 已持未使用同类券）、已忽略（会员ID 重复自动去重）、待执行（校验通过）；结果表展示全部明细，仅「待执行」进入发放 / 审核。

3.2 发券审核（V1后台 · 改造）

发券并入通用「后台操作审核记录」，操作类型 = 人工充值、子类型新增「充值优惠券」，其余字段复用；是否审核由系统按操作人账号权限自动判定，需审核账号点「提交审核」入审核流。

审核结果：通过 → 发券（调发券服务）；拒绝 → 不发放、仅留痕。审核记录支持按 操作类型 / 子类型 / 审核状态 / 操作人 / 审核人 筛选与导出。

3.3 客户端（新增 · 多语言）

弹窗领取（登录首页弹窗领券）/ 我的充值优惠券（券包）/ 充值用券 / 个人中心；均支持多语言（当前英 / 印地 / 越南，可扩展）。

3.4 赠金计算规则（券 vs 通道赠金）

本节是券赠金与通道赠金的唯一计算口径，#31 / #16 / #20 三份需求共用；其余章节与业务规则只引用本节、不重复算式。

① 输入量

符号含义
A充值本金（门槛 / 比例 / 手续费都基于它）
r / cap / th券比例 / 券上限 / 券门槛（th = 0 表示无门槛）
chPct1 / chPct2 / chPct3三类通道奖励费率：充值通道赠金 / VIP等级充值奖励 / 快捷金额充值奖励
feePct手续费率
m_券 / m_通道打码倍数（券 / 通道）

② 算式

步骤算式
① 券可用性usable = (A >= th) —— 含等于
② 券赠金cpAward = min( floor(A × r), cap ) —— 先取整、再封顶
③ 通道候选ch1 = floor(A × chPct1) —— 充值通道赠金
ch2 = floor(A × chPct2) —— VIP等级充值奖励
ch3 = floor(A × chPct3) —— 快捷金额充值奖励
④ 关系裁决见下方「③ 三种关系」表
⑤ 手续费fee = floor(A × feePct)
⑥ 实际上分youGet = A + award − fee

取整一律 floor（向下取整），精度到整数 ₹；四个候选（cpAward / ch1 / ch2 / ch3）各自独立取整后再参与裁决（不是先加后取整）。

③ 三种关系

关系算式券消耗明细
互斥·取高
（默认）		award = max(cpAward, ch1, ch2, ch3)，平局取券券赢 → 消耗；券落败 → 不消耗、退回券包，「已使用」不计数1 行 + 来源标注
叠加award = cpAward + ch1 + ch2 + ch3（四者全给）消耗2 行：「充值奖励」(= ch1+ch2+ch3) ＋「充值优惠券彩金」(= cpAward)；铁律 —— 逐行相加 = award
互斥·优先券award = cpAward，无条件用券、完全不比通道消耗1 行 + 来源标注

④ 打码量（各算各的）：互斥 → award × m_赢家倍数；叠加 → cpAward × m_券 + (ch1+ch2+ch3) × m_通道。

⑤ 来源标注文案（明细为 1 行时必须带）：

四、业务规则
BR-01
存款类型选「充值优惠券」：隐藏 存款金额 / 打码量，显示 券规格 + 发放方式。
BR-02
发放方式仅「充值优惠券」有：自动领取＝券直接入袋 + 通知；弹窗领取＝会员登录首页弹窗领取才入袋。
BR-03
彩金充值 / 人工充值 / USDT 直接到账、彩金扣除为扣款 → 均无发放方式（非券的资金操作没有"领取"概念）。
BR-04
是否审核由运营管理里「操作人后台账号」的权限决定、系统自动判断（不在发券表单手动勾选）；需审核时仅按数量阈值触发（如批量 > N 人），金额不作为判定条件。
BR-05
券来源：券码库＝唯一券源；现填规格也自动登记成券码。批量时名单只管发给谁（会员ID），券规格 / 发放方式在弹窗统一配置。
BR-06
额度占用（弹窗领取）：此处「额度」= 券码发放上限的次数配额；弹窗领取发放时先占用 1 个名额、会员领取才确认消耗，未领作废 / 审核拒绝 → 名额退回、可再发他人；券码未设上限（0=不限）则无占用，自动领取即时消耗。
BR-07
券 vs 通道赠金关系：互斥·取高（默认）/ 叠加 / 互斥·优先券，三选一（属券规格 —— 从券码库选时随券只读带出，现填规格时录入）。计算口径见「3.4 赠金计算规则（券 vs 通道赠金）」，本条不另立算式。
五、限制条件与校验
  • 批量单次 ≤ 1.5 万条，文件 ≤ 4M（xlsx / csv）。
  • 校验：黑名单 / 已持未使用同类券（同规格 / 同类型） → 失败；会员ID 重复 → 自动去重忽略；单次仅限同一充值类型。单会员发放同样校验黑名单 / 同类券。
  • 现填规格须做合法区间校验（比例% / 上限₹ / 已领取·未领取券有效期 / 打码量倍数），避免录入异常规格。
  • 弹窗领取须设「领取截止」（默认 7 天、运营可自定义；0 = 不限制，发放后超期未领作废、额度退回）。
  • 批量失败 / 忽略清单沿用现有「批量充值」的失败处理逻辑（逐行跳过 + 单条隔离 + 结果清单带状态与原因，复用现有流程；不含失败清单导出 / 修正后重传）。
  • 口径：会员主动获取（红包码兑换 / 注册得券）一律自动入袋；仅充值优惠券定向发可选弹窗领取。
