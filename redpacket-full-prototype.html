一、名词术语
术语定义
红包码代理派发的兑换码，用户输码兑换后得到现金或充值优惠券。
奖励类型红包发放的奖励种类：现金 / 充值优惠券（本期新增）。
接受范围红包发给谁：代理维度（下级）或子代理维度（本代理下级的指定子代理ID），二选一互斥。子代理维度按子代理ID 逐个枚举，区别于粗粒度的「全部下级 / 单个直属代理」。
领取者 = 被指定的子代理本人（代理体系中代理亦是会员，故其以会员身份兑换、持券并在充值时使用），非其下级会员。
发券前置授权给指定代理配置可发的充值优惠券码 + 可用次数（配额）。次数有两个来源、两者累加：
① 创建券码时：在 V1 后台建券码时绑定到指定代理并配一个次数；
② 会员列表·红包充值：运营在该页给代理追加某券码的次数。
两处次数累加（不是覆盖） = 该代理该券码的总可发次数。此后代理在「红包派送」创建红包时，「选择充值优惠券码」下拉仅列出授权给他的券码，并在剩余次数范围内发放。
券规格（只读）券的比例 / 上限 / 已领取券有效期 / 未领取券有效期（过期时间）/ 门槛 / 券 vs 通道赠金关系，均取自 V1 后台券码库、由运营创建券码时设定，代理不可改。
券 vs 通道赠金关系券规格的一项配置，决定用券充值时券赠金与通道赠金谁生效、是否相加，三选一：互斥·取高（两者只取金额高的一方，平局取券，券落败则不消耗、退回券包）/ 叠加（两者全给、相加）/ 互斥·优先券（无条件只给券赠金、完全不比通道）。计算口径见「3.7 赠金计算规则（券 vs 通道赠金）」。
红包码状态红包码的生命周期：进行中 / 已抢完（剩余数量=0）/ 已取消（手动取消）/ 已过期（券红包超过未领取券有效期自动置）。

端页面 / 模块类型核心内容

3.7 赠金计算规则（券 vs 通道赠金）

本节是「券赠金」与「通道赠金」的唯一口径，#31 / #16 / #20 三份需求共用；其余章节与业务规则只引用本节、不重复。

业务流程：用户用券充值时，实际到账 = 实付金额 ＋ 赠金 − 手续费。「赠金」由两部分竞争或叠加得到——券赠金（这张券带来的）与通道赠金（充值方式自带的活动奖励）；
二者谁生效、是否相加，由券规格里配置的赠金关系决定（三选一，运营建券时设定，代理 / 会员不可改）。

字段定义：

概念说明
实付金额用户本次实际充值金额；
券门槛、券比例、手续费均以它为基数（不以到账额为基数）
券赠金实付金额按券比例计算，不超过券上限；
未达券门槛则该券不可用
通道赠金充值方式自带的活动奖励，含三类：充值通道赠金 / VIP等级充值奖励 / 快捷金额充值奖励
手续费按实付金额的固定费率扣取
打码量赠金到账后、提现前需完成的流水倍数；
券赠金与通道赠金各自按自己的倍数计算
取整规则券赠金、通道赠金、手续费均向下取整为整数；
展示统一保留 2 位小数（如 50 显示为 50.00），各自单独计算

三种赠金关系：

赠金关系含义券是否消耗到账明细
互斥·取高
（默认）		券赠金与三类通道奖励中只发金额最高的一个；
金额相等时优先发券		券胜出 → 消耗；
券未胜出 → 不消耗、退回券包，不计入「已使用」		一行「奖励」＋来源标注
叠加券赠金与三类通道奖励全部发放、金额相加消耗两行：「充值奖励」（三类通道合计）＋「充值优惠券彩金」；
两行金额相加 = 总赠金
互斥·优先券无条件只发券赠金，不与通道比较（即使通道更高）消耗一行「奖励」＋来源标注

来源标注：明细为一行时（取高 / 优先券）必须标注赠金来源，让会员看清钱从哪来、券有没有被扣：

  • 券胜出：奖励 +₹50（充值优惠券 10%，高于通道 ₹45）
  • 券未胜出：奖励 +₹90（VIP等级奖励 高于券 ₹50；券未使用、已退回）

到账 = 实付金额 ＋ 赠金 − 手续费（手续费 = 实付金额 × 2%）。
例：充 ₹500 取高，到账 500 ＋ 50 − 10 = ₹540；
充 ₹1000 取高，到账 1000 ＋ 90 − 20 = ₹1070。可作验收基准。

四、业务规则
BR-01
红包码可发充值优惠券：券进用户「我的充值优惠券」，使用与到账复用现有充值优惠券逻辑。
BR-02
接受范围 = 单选：代理维度（直属 / 佣金 / 全部下级 / 指定直属代理）或子代理维度（指定 / 导入子代理ID），二选一互斥。
BR-03
子代理维度按「子代理ID 精准匹配」——按 ID 逐个枚举指定子代理（区别于粗粒度的全部下级 / 单个直属代理）；
指定 / 导入的子代理ID 必须是本代理的下级（按「代理-下级」归属校验，非下级 ID 自动剔除）；
支持手填 ID（逗号 / 中文逗号 / 顿号 / 空格 / 换行分隔）与批量导入 CSV / Excel。领取者为被指定的子代理本人（其以会员身份兑换、持券、用券充值），而非该子代理的下级会员。
BR-04
券 vs 通道赠金关系：互斥·取高（默认）/ 叠加 / 互斥·优先券，三选一；
在 V1 后台创建充值优惠券码时配置（属券规格、代理不可改）。计算口径见「3.7 赠金计算规则（券 vs 通道赠金）」，本条不另立算式；
充值页的明细展示形态见 BR-07。
BR-05
运营前置授权：代理只能在授权额度 / 次数内派券，券规格由运营在券码库设定、代理不可改。
BR-06
红包码状态与取消：状态分 进行中 / 已抢完（剩余数量=0 自动置）/ 已取消（手动「取消红包码」）。取消或抢完后立即停止发放（新用户不能再领 / 兑换），但已领到用户手里的红包 / 券不受影响、可正常使用与到账；
已抢完 / 已取消的码不可再次发放。券红包另有 「已过期」：过期时间 = 创建充值优惠券时配置的未领取券有效期，到期自动置过期，行为同已取消（停止发放 / 兑换，已领不受影响）。
BR-07
充值页「实际上分」明细展示（随赠金关系动态）：叠加时，充值优惠券彩金与通道充值奖励都到账，明细拆成「充值奖励」+「充值优惠券彩金」两条分别相加（如 500+45+50−10 = 585）；
互斥·取高 / 互斥·优先券时只有一个生效，合并为一条「奖励」+ 来源标注（如 500+50−10 = 540）。铁律：每一行都必须真正计入实际上分，逐行相加 = 实际上分，互斥时不得并列两条「+」以免误导。各行金额的计算口径见「3.7 赠金计算规则（券 vs 通道赠金）」——本条只管展示形态，不另立算式（上述 585 / 540 两例即 3.7 算例表 ₹500 行的「叠加」与「取高」结果）。
BR-08
授权次数扣减与退回：券红包在发放（创建）时按发放数量一次性预扣授权次数（复用现有红包「发放预扣」机制）；
红包码取消 / 过期后，未被领取的次数退回代理授权额度。现金红包沿用现有逻辑：发放时预扣全额、取消不退回（维持现状不动）。
BR-09
多语言：客户端（输码兑换 / 我的充值优惠券 / 充值用券 / 个人中心）与代理后台（红包派送 / 领取记录 / 赠金记录等）均按现有多语言机制支持多语言（默认中文，可切 英 / 印地 / 越南…）。
BR-10
券码停用联动：在 V1 后台「券码列表」停用某充值优惠券码后，选用该券码的红包码同步失效（停止发放 / 兑换，行为同已取消 / 已过期）；
已领到用户手里的券不受影响、可正常使用与到账，未领次数按 BR-08 退回代理配额。（授权无需单独"撤销"——停用券码即达到停用其红包码的效果。）
五、限制条件与校验
  • 红包数量 ≤ 5 万个。
  • 派发数量 ≤ 授权剩余次数（超出不可发）。
  • 校验：子代理ID 重复 → 自动去重（保留一条，不报错）。
  • 校验：被指定的子代理账号处于黑名单，或已持有同一张券 → 该 ID 校验失败、不发放（计入「剔除 N 个」提示）。
  • 单次限同一奖励类型（一个红包码不可混发现金与充值优惠券）。
  • 红包码限领走子代理维度（子代理ID 名单），不走手机号池。
  • 每人限领沿用现有：固定码每人限 1 次、随机码一码限 1 次（无需新增）。
  • 充值页每次充值只能使用 1 张充值优惠券（不叠加多张券）。
