红包码充值优惠券系统
Red-Packet Coupon · 代理红包码可发充值优惠券 + 会员定向发放
一、需求背景与目标
1.1 背景与痛点
  • 代理红包只能发现金，无法发充值优惠券，拉新 / 促活钩子单一。
  • 会员定向发券缺少手段：原红包接受范围只有代理下级维度，无法精准发给指定下级会员。
1.2 需求目标
  • 红包码可发充值优惠券，券进「我的充值优惠券」，充值时使用。
  • 接受范围新增会员维度（指定子代理ID / 导入会员ID）。
  • 运营前置授权：代理只能在授权额度 / 次数内派券，券规格由运营设定、代理不可改。
二、名词术语
术语定义
红包码代理派发的兑换码，用户输码兑换后得到现金或充值优惠券。
奖励类型红包发放的奖励种类：现金 / 充值优惠券（本期新增）。
接受范围红包发给谁：代理维度（下级）或会员维度（本代理下级的指定子代理ID），二选一互斥。
发券前置授权给指定代理配置可发的充值优惠券码 + 可用次数（配额）。次数有两个来源、两者累加：
① 创建券码时：在 V1 后台建券码时绑定到指定代理并配一个次数；
② 会员列表·红包充值：运营在该页给代理追加某券码的次数。
两处次数累加（不是覆盖） = 该代理该券码的总可发次数。此后代理在「红包派送」创建红包时，「选择充值优惠券码」下拉仅列出授权给他的券码，并在剩余次数范围内发放。
券规格（只读）券的比例 / 上限 / 已领取券有效期 / 未领取券有效期（过期时间）/ 门槛 / 券 vs 通道赠金关系，均取自 V1 后台券码库、由运营创建券码时设定，代理不可改。
券 vs 通道赠金关系用券充值时券赠金与通道赠金的计算关系：互斥·取高 / 叠加 / 互斥·优先券。
红包码状态红包码的生命周期：进行中 / 已抢完（剩余数量=0）/ 已取消（手动取消）/ 已过期（券红包超过未领取券有效期自动置）。

四、功能详述
4.1 红包派送（代理端 · 改造）

现金红包与充值优惠券红包同一列表，按「奖励类型」区分 / 筛选；点「＋新增」打开创建红包弹窗。列表含「已抢完」状态列与「取消红包码」操作（状态与取消规则见 BR-06）。

创建红包弹窗字段：

字段必填说明 / 取值
奖励类型是现金 / 充值优惠券（新）
选择充值优惠券码条件奖励类型=充值优惠券时必填，下拉仅列运营前置授权的券码；选后展示券规格（只读）
接受范围是代理维度 或 会员维度（指定子代理ID / 导入会员ID），二选一互斥
接收条件否存款 / 投注 / 亏损（多少天内或历史累计）；判定时机与阈值录入沿用现有红包逻辑
数量 / 金额类型是红包个数、固定 / 随机金额（随机金额沿用现有红包逻辑；券红包为按张发）
4.2 接受范围（改造）

在原「代理下级维度」（直属下级 / 佣金下级 / 全部下级 / 指定直属代理）基础上，新增会员维度：指定子代理ID（手填，逗号分隔）/ 导入会员ID（CSV），也可从会员白名单库选名单。所填 / 导入的会员ID 必须属于本代理的下级（非任意会员）——系统按「代理-下级」归属校验，非本代理下级的 ID 自动剔除、不发放；校验后提示「剔除 N 个（非下级 / 黑名单 / 重复）、有效 M 个」，仅对有效 ID 发放。

4.3 会员列表·红包充值（V1后台 · 改造）

运营在会员列表对单个代理操作，配置项与新增列如下：

配置 / 项类型说明
红包充值（现金）配置给代理充现金红包额度（红包余额），用于派发现金红包
前置授权充值优惠券码 + 总次数配置给代理追加某券码的可发次数；与「创建券码时」配的次数累加（不覆盖）（见名词术语「发券前置授权」 / BR-08）
红包 / 券 剩余额度新增列一列上下两行：红包剩余金额（₹） + 充值优惠券码剩余次数（张）
生效前提规则代理获授权后，才能在「红包派送」下拉选到该券码、派发对应券红包
4.4 领取记录·效果 / 赠金记录（代理端 · 改造）

领取记录·效果：红包领取统计 + 转化效果（核销率等）。赠金记录：代理赠金账变，现金红包扣款、券红包零扣（券彩金成本由平台承担，统一计入账变「充值优惠券奖励」，代理后台暂不体现）。

4.5 客户端（新增 · 多语言）

输码兑换（输红包码得券）/ 我的充值优惠券（券包）/ 充值用券 / 个人中心；均支持多语言（当前英 / 印地 / 越南，可扩展）。

个人中心「充值」按钮气泡：动态展示当前可享的最高充值奖励（如「充 ₹500 返 ₹50」），随可用优惠实时更新。换算规则：从会员当前所有可用充值优惠券 / 通道充值奖励中取彩金上限最高的一档（单档，不算叠加组合），按 Y（返）= 该档彩金上限 ₹C、X（充）= max( ⌈C ÷ 比例%⌉ , 门槛 ) 换算成「充 X 返 Y」。示例：10% · 上限₹50 · 无门槛 → 充 ₹500 返 ₹50；20% · 上限₹30 · 满₹200 → 充 ₹200 返 ₹30。多档时先比 Y 取最大、Y 相同再取 X 最小（更易达成）；无上限的奖励不进气泡、无可用奖励不展示气泡。「我的充值优惠券」入口带 NEW 标识、可点击进入券包。

五、业务规则
BR-01
红包码可发充值优惠券：券进用户「我的充值优惠券」，使用与到账复用现有充值优惠券逻辑。
BR-02
接受范围 = 单选：代理维度（直属 / 佣金 / 全部下级 / 指定直属代理）或会员维度（指定 / 导入会员ID），二选一互斥。
BR-03
会员维度按「会员ID 精准匹配」；指定 / 导入的会员ID 必须是本代理的下级（非任意会员，按「代理-下级」归属校验，非下级 ID 自动剔除）；支持手填ID / 批量导入 CSV / 从会员白名单库选名单。
BR-04
券 vs 通道赠金关系：互斥·取高（默认）/ 叠加 / 互斥·优先券；在 V1 后台创建充值优惠券码时配置（属券规格、代理不可改），充值页按此计算与展示（见 BR-07）。
BR-05
运营前置授权：代理只能在授权额度 / 次数内派券，券规格由运营在券码库设定、代理不可改。
BR-06
红包码状态与取消：状态分 进行中 / 已抢完（剩余数量=0 自动置）/ 已取消（手动「取消红包码」）。取消或抢完后立即停止发放（新用户不能再领 / 兑换），但已领到用户手里的红包 / 券不受影响、可正常使用与到账；已抢完 / 已取消的码不可再次发放。券红包另有 「已过期」：过期时间 = 创建充值优惠券时配置的未领取券有效期，到期自动置过期，行为同已取消（停止发放 / 兑换，已领不受影响）。
BR-07
充值页「实际上分」明细展示（随赠金关系动态）：叠加时，充值优惠券彩金与通道充值奖励都到账，明细拆成「充值奖励」+「充值优惠券彩金」两条分别相加（如 500+45+50−10 = 585）；互斥·取高 / 互斥·优先券时只有一个生效，合并为一条「奖励」（如 500+50−10 = 540）。铁律：每一行都必须真正计入实际上分，逐行相加 = 实际上分，互斥时不得并列两条「+」以免误导。
BR-08
授权次数扣减与退回：券红包在发放（创建）时按发放数量一次性预扣授权次数（复用现有红包「发放预扣」机制）；红包码取消 / 过期后，未被领取的次数退回代理授权额度。现金红包沿用现有逻辑：发放时预扣全额、取消不退回（维持现状不动）。
BR-09
多语言：客户端（输码兑换 / 我的充值优惠券 / 充值用券 / 个人中心）与代理后台（红包派送 / 领取记录 / 赠金记录等）均按现有多语言机制支持多语言（默认中文，可切 英 / 印地 / 越南…）。
BR-10
券码停用联动：在 V1 后台「券码列表」停用某充值优惠券码后，选用该券码的红包码同步失效（停止发放 / 兑换，行为同已取消 / 已过期）；已领到用户手里的券不受影响、可正常使用与到账，未领次数按 BR-08 退回代理配额。（授权无需单独"撤销"——停用券码即达到停用其红包码的效果。）
六、限制条件与校验
  • 红包数量 ≤ 5 万个。
  • 派发数量 ≤ 授权剩余次数（超出不可发）。
  • 校验：会员ID 重复 → 自动去重忽略；黑名单 / 已持同券 → 失败；单次限同一奖励类型。
  • 红包码限领走会员维度（会员ID / 分组）。
  • 每人限领沿用现有：固定码每人限 1 次、随机码一码限 1 次（无需新增）。
  • 充值页每次充值只能使用 1 张充值优惠券（不叠加多张券）。
