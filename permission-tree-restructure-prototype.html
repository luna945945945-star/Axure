一、需求背景 & 目标

  • 痛点：给角色授权时，权限树在右侧约 30% 窄栏（≈380px）逐行竖排——一个菜单的多个权限要逐行勾，一屏只看到 2~3 个菜单，容易误勾、漏勾。
  • 目标：一屏看到多个菜单的权限，整组扫视、方便勾选。
  • 硬约束（已与开发确认）：只改客户端样式 / 承载容器，不改勾选逻辑与数据结构，原勾选体系保持不变。

二、最终方案（已确认）：权限编辑 → 抽屉放大 + 紧凑平铺

点角色行的「权限编辑」→ 抽屉式从右展开放大（≈82vw），权限树紧凑平铺：模块/菜单/页面成行，权限以对齐网格平铺在「页面」标签右侧（借鉴密集布局，配合对齐网格更整洁）；点 ✕收起 / 遮罩 / ESC 收起。

角色列表→ 权限编辑 抽屉放大编辑→ 勾选/搜索 保存→ 含敏感? 敏感二次确认→ 确定 成功 · 收起

※ 不含敏感权限时跳过二次确认直接保存；敏感判定与弹窗为现有逻辑，原样保留。

当前系统敏感项仅 2 处：系统管理/系统用户页面 · 编辑用户可USDT出款权限（叶子级）、财务管理/支代付三方管理页面（页面级）。

三、🔧 开发改动点

总原则：只动「渲染样式 + 承载容器」，不动「数据 + 勾选逻辑」。新增/改 禁止改

① 入口与容器

  • 改「权限编辑」点击 → 打开 el-drawer（宽 ≈82vw，带遮罩）；权限树整体渲染进抽屉 body（容器迁移，组件实例与勾选逻辑不变）。
  • 新增抽屉头：分配角色名 + 收起全部/展开全部/全选/保存/✕收起；支持 遮罩点击 / ESC 收起。
  • 改移除原右侧常驻窄栏，角色列表恢复整宽。

② 紧凑平铺 + 选中样式（样式）

  • 改权限横向平铺：把「页面」节点设为弹性行——页面标签 居左定宽，其 .el-tree-node__children 移到右侧做 对齐网格（grid auto-fill）自动换行；模块/菜单仍整行并显示「已选/总」计数。
  • 改选中样式仅保留复选框打勾：去掉旧的整行高亮/背景填充。
  • 改收紧行距、去卡片重边框（改用浅虚线分隔），ellipsis+title 处理长名 → 同屏更多。
  • 改「敏感」标签随节点带入：叶子权限的敏感标显示在该权限名后；页面级敏感标显示在页面标签处（红色）。标签数据来源不变。

③ ⛔ 明确不许改（回归基线）

  • 禁改权限数据源、节点 node-key、check-keys / v-model。
  • 禁改父子联动勾选、半选(indeterminate) 计算、搜索 filter-node-method、全选、收起全部。
  • 禁改「保存权限」提交结构与入参字段（后端无感）。
  • 禁改保存「敏感权限」二次确认弹窗（敏感权限提醒）：触发点从旧面板按钮移到抽屉「保存权限」，敏感判定 + 弹窗内容 + 取消/确定逻辑完全不变；敏感标记来源不变。

四、🧪 测试点 & 回归清单

A. 逻辑回归（必须与改造前一字不差）

  • 勾单个权限 → 所属 页面/菜单/模块 三级父节点呈半选(─)
  • 勾页面/菜单/模块 → 旗下权限全部选中；再点 → 全部取消
  • 取消某页最后一个已选权限 → 三级父级 半选→未选 正确回落
  • 全选 / 收起全部 / 展开全部、模块/菜单/页面「已选/总」计数正确
  • 搜索菜单名、权限名：命中显示+高亮，非命中隐藏；清空恢复
  • ★ 保存权限：抓包对比改造前后提交的权限 key 集合完全一致
  • 编辑已有角色：已授权权限回显勾选、父级半选正确；切换角色刷新对应权限

B. 抽屉交互 & 紧凑平铺（新增）

  • 点「权限编辑」→ 抽屉从右展开放大，标题与分配角色名正确
  • 抽屉内 勾选 / 搜索 / 全选 / 收起全部 / 保存 均正常
  • ✕收起 / 点遮罩 / 按 ESC → 抽屉收起；未保存改动关闭处理符合产品约定
  • 选中样式仅复选框打勾，无整行高亮
  • 权限网格随分辨率自适应换行（1366/1440/1920）；22 权限页面显示紧凑无错位
  • 长权限名截断省略 + 悬浮 title 显示全名

C. 敏感权限二次确认（现有逻辑，重点回归）

  • 勾选含「敏感」的权限/页面 → 点保存 → 弹「敏感权限提醒」二次确认窗
  • 弹窗按 模块/菜单/页面 列出敏感操作（叶子级 & 页面级都覆盖），内容与改造前一致
  • 点「取消」→ 不保存；点「确定」→ 才真正提交保存（入参一致）
  • 未勾选任何敏感项 → 直接保存、不弹二次确认
  • 「敏感」标签在 改造前竖排 / 改造后平铺 均正确显示（叶子在权限名后、页面在页面标签处）
